## Sneltoets-register (architectuur — enige bron van waarheid)
Nieuw bestand `src/hooks/keyboard/shortcutRegistry.ts`. Alle sneltoetsen (bestaand + nieuw) staan hier
als declaratieve lijst; `useKeyboardShortcuts` matcht ertegen en de Ctrl+/-dialoog rendert eruit. Doel
(user-eis): een toets toevoegen = **één entry + één i18n-key**, verder niets.

```ts
interface ShortcutDef {
  id: string;                       // stabiel, bv. 'edit.task'
  combo: {                          // Cmd op macOS = Ctrl hier (mod: true)
    key: string;                    // 'F2' | 'Insert' | 'a' | 'ArrowUp' | '/' | 'Home' ...
    mod?: boolean;                  // Ctrl (Win/Linux) of Cmd (mac)
    shift?: boolean; alt?: boolean;
  };
  category: 'file' | 'edit' | 'structure' | 'view' | 'nav';
  labelKey: string;                 // i18n-key voor de overzichtsdialoog
  run: (store: AppStore) => void;   // roept bestaande store-acties aan
  allowInInput?: boolean;           // zoals Ctrl+S/F5/F11: werkt óók in invoervelden
  when?: () => boolean;             // optionele extra-guard (bv. 'pure boommodus' voor indent)
  displayOnly?: boolean;            // in dialoog tonen maar niet zelf afhandelen (bv. Gantt +/- die in
                                    //   useZoomShortcuts leeft) — geen dubbele handler
}
```

Migratie: **alle** bestaande globale sneltoetsen uit `useKeyboardShortcuts.ts` verhuizen naar het register
(gedrag byte-identiek houden — undo/redo, save-whitelisting, Ctrl+1..9 documentwissel, zoom, presentatie).
Families als `Ctrl+1..9` mogen één entry met een key-range-matcher zijn, of 9 gegenereerde entries — kies
wat het register schoon houdt. `useZoomShortcuts.ts`-toetsen (`+`/`-`/`0`/`Ctrl+0`) blijven functioneel
waar ze zijn, maar krijgen `displayOnly`-entries zodat de overzichtsdialoog compleet is (geen dubbele
afhandeling). Match-helper respecteert de bestaande typing-guards (niet vuren in input tenzij `allowInInput`).

## Nieuwe store-acties / helpers (golf 1 — geen UI)
Alle nieuw en **undo/redo-integratie zoals de omliggende taak-acties** (zelfde history-wrapper als
`addTask`/`moveTask`/`updateTask`). WBS/nummering consistent houden zoals de bestaande acties dat doen.

1. `reorderSibling(taskId, direction: 'up'|'down')` — verwissel `taskId` met zijn directe buur binnen
   `parent.childIds` (top-level: binnen de root-lijst). No-op aan de rand. Alleen array-herschikking;
   raakt GEEN tijden/CPM. Bestand: `src/state/slices/taskSlice.ts` (+ type in `types.ts`).
2. `addTask`-uitbreiding: optionele plaatsing `{ position?: { anchorId: string, where: 'above'|'below' } }`.
   Zonder → huidig gedrag (achteraan `childIds`). Met → invoegen op de juiste index binnen dezelfde
   ouder als `anchorId`. Gebruikt door `Insert` en de contextmenu-items "Invoegen boven/onder".
   Wijzig de bestaande callers NIET (default-gedrag identiek).
3. `selectAllTasks()` — zet `selectedTaskIds` op alle **zichtbare** taak-ids (respecteer ingeklapte
   groepen: gebruik dezelfde zichtbaarheids-afleiding als de renderer/`visibleRows`).
4. `expandAll()` / `collapseAll()` — itereer over alle summary-taken (`childIds.length>0`) en zet
   `collapsed` uniform (niet togglen: expliciet true/false). Voor het bandkop-contextmenu.
5. `computeScrollToDate(date, state)` in `src/utils/ganttViewport.ts`, naar analogie van
   `computeFitToProject`: bereken de `scrollX` zodat `date` (default: `project.statusDate` of vandaag)
   links met kleine marge in beeld komt; zoom onaangeroerd. Wordt door `Ctrl+Home` aangeroepen.

## Box-selection (golf 4)
Sleep een rechthoekig kader om meerdere taken tegelijk te selecteren. Selectie op **rij-intersectie**:
elke taak waarvan de rij-band verticaal door het kader wordt geraakt, wordt geselecteerd (horizontale
tijd-as doet niet mee — voorspelbaar en werkt zowel in de takentabel als in het chart).

**Gebaar (botst niet met bestaande drags in `GanttCanvas.handleMouseDown`):**
- Start op **lege achtergrond** (geen bar-hit, niet op de splitter, onder de header).
- **Takentabel** (`x < taskTableWidth`): lege-drag is daar nu vrij (pant nooit) → altijd box-select.
- **Chart-gebied** (`x >= taskTableWidth`): box-select **behalve** wanneer `scrollMode==='drag'` (dan
  blijft lege-drag pannen zoals nu — pan wint, ongewijzigd gedrag). In de overige scroll-modi = box-select.
- `Ctrl/Cmd`+sleep = **toevoegen** aan bestaande selectie; kale sleep = **vervangen**. `Shift`+drag blijft
  gereserveerd voor relatie-tekenen vanaf een bar (ongemoeid).
- Drempel: pas als de sleep > ~4px is → kader; anders val terug op de bestaande klik-afhandeling
  (geen per ongeluk lege selectie bij een gewone klik).

**Rendering:** een half-transparant selectie-rechthoekje tijdens de drag (overlay/canvas), opgeruimd bij
mouseup. Nieuwe lokale state (bv. `boxSelectState`) naar analogie van `panState`/`depDragState`; window-
mousemove/up-listeners zoals de bestaande drag-effects. Selecteren via `selectTask(id, additive)` per
geraakte taak, of een kleine bulk-set. Raakt GEEN tijden/CPM.

**Verificatiepoort golf 4:** `tsc` schoon · `npm run build` groen · QA: kader in takentabel én in chart
(beide scroll-modi), plain vs Ctrl-additief, geen conflict met pannen/bar-drag/relatie-tekenen, geen
per-ongeluk-deselectie bij een gewone klik.
