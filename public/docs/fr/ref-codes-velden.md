# Codes et champs (boîte de dialogue de structure)

La fenêtre **Codes et champs** gère les définitions de structure du projet : les **codes d'activité** (dimensions librement définissables telles que Localisation ou Discipline) et les **champs personnalisés** (champs utilisateur typés). Les valeurs par tâche se remplissent ensuite via le panneau des propriétés ou la [boîte de dialogue de tâche](docs://ref-taakdialoog).

## Ouverture

**Planification** → groupe de ruban **Structure** → **Codes et champs**. **Échap**, la croix de fermeture ou un clic en dehors de la fenêtre la ferme. Toutes les modifications prennent effet immédiatement (et peuvent être annulées avec Ctrl+Z) — il n'y a pas de bouton d'enregistrement séparé.

## Codes d'activité

« Dimensions librement définissables (p. ex. Localisation, Discipline) pour grouper et filtrer — au plus une valeur par type et par tâche. »

Un bloc par type de code :

- **Nom du type de code** — modifiable directement.
- **Supprimer le type de code** (corbeille) — supprime le type, y compris toutes les valeurs et les attributions sur les tâches.
- Une ligne par valeur : **Code** (libellé court), **Description** et un sélecteur de **Couleur** (colore entre autres les regroupements), plus un bouton de suppression.
- **Ajouter une valeur** — nouvelle valeur sous ce type.

En bas : champ de saisie **Nouveau type de code (p. ex. Localisation)** + bouton **Ajouter un type de code** (Entrée fonctionne aussi).

## Champs personnalisés

« Champs utilisateur typés, affichés comme colonnes dans le tableau et modifiables par tâche. »

Une ligne par champ : le **nom** (modifiable directement), le **type** (lecture seule après création) et un bouton de suppression.

En bas : champ de saisie **Nouveau champ (p. ex. Entrepreneur)**, un sélecteur de type — **Texte**, **Nombre**, **Nombre entier**, **Coût**, **Date** ou **Oui/non** — et le bouton **Ajouter un champ** (Entrée fonctionne aussi). Le type ne peut pas être modifié après la création ; créez un nouveau champ si nécessaire.

## Où apparaissent les définitions

- Comme section d'entrée **Codes et champs** par tâche dans le panneau des propriétés et la boîte de dialogue de tâche.
- Comme colonnes dans la vue tableau (champs personnalisés) et comme dimension de regroupement/filtre (codes d'activité).
