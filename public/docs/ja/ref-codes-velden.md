# コードとフィールド（構造ダイアログ）

**コードとフィールド** ウィンドウは、プロジェクトの構造定義を管理します：**アクティビティコード**（場所や工種のような自由に定義できる分類軸）と **カスタムフィールド**（型付きのユーザーフィールド）です。タスクごとの値は、プロパティパネルまたは [タスクダイアログ](docs://ref-taakdialoog) から入力します。

## 開く

**計画** → **構造** リボングループ → **コードとフィールド**。**Esc**、閉じるボタン、またはウィンドウの外側のクリックで閉じます。すべての変更は即座に反映され（Ctrl+Z で取り消せます）、独立した保存ボタンはありません。

## アクティビティコード

「自由に定義できる分類軸（例：場所、工種）。グループ化とフィルタリングに使用でき、タスクごと・タイプごとに値は最大 1 つです。」

コードタイプごとに1つのブロック：

- **コードタイプ名** — 直接編集できます。
- **コードタイプを削除**（ゴミ箱）— そのタイプを、すべての値およびタスクへの割り当てごと削除します。
- 値ごとに1行：**コード**（短いラベル）、**説明**、**色** ピッカー（グループ化などの色分けに使用）、削除ボタン。
- **値を追加** — このタイプの下に新しい値を追加します。

下部：入力フィールド「**新しいコードタイプ（例：場所）**」＋ボタン **コードタイプを追加**（Enter でも可）。

## カスタムフィールド

「型付きのユーザー定義フィールドで、テーブルに列として表示され、タスクごとに編集できます。」

フィールドごとに1行：**名前**（直接編集できます）、**種類**（作成後は読み取り専用）、削除ボタン。

下部：入力フィールド「**新しいフィールド（例：施工業者）**」、種類ピッカー — **テキスト**、**数値**、**整数**、**コスト**、**日付**、**はい/いいえ** — と **フィールドを追加** ボタン（Enter でも可）。種類は作成後に変更できません。必要な場合は新しいフィールドを作成してください。

## 定義がどこに表示されるか

- プロパティパネルとタスクダイアログの、タスクごとの **コードとフィールド** エントリセクションとして。
- テーブルビューの列（カスタムフィールド）、およびグループ化／フィルターの軸（アクティビティコード）として。
