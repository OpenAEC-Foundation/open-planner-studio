# 計画と WBS

スケジュールはタスク構造から始まります。どのようなタスクが存在し、それらはどうフェーズに分解され、どの瞬間がマイルストーンに値するほど重要なのでしょうか。このガイドでは、[クイックスタート](docs://quick-start) ガイドよりもさらに踏み込んでこの基礎を扱います — ここでは *どうやって* インデントするかだけでなく、サマリータスクが実際に何をするのか、3種類のマイルストーンがどう違うのか、タスクに独自のコードやフィールドをどう持たせるか、タスクごとのメモをどう管理するかを学びます。

## ここで学ぶこと

- インデントとサマリータスクを使ったタスク構造（WBS）の構築。
- 再インデントせずに、同じレベル内でタスクを移動する方法。
- 3種類のマイルストーンと、契約上の重要な瞬間のための独立した必須フラグ。
- **コードとフィールド** ウィンドウでのアクティビティコードとカスタムフィールドの管理、およびそれらによるグループ化。
- メモ（タスクごとのチェックリスト）を使って未対応の項目を管理する方法。

完全なサンプルを見ながら進めたい場合は、**ファイル → サンプル** から [Verbouwing & Aanbouw Eengezinswoning](examples://showcase-verbouwing-eengezinswoning.ifc) を開いてください — フェーズ分け「1. Voorbereiding」（準備）／「2. Fundering & ruwbouw」（基礎・躯体工事）／「3. Afbouw」（仕上げ）／「4. Oplevering」（引き渡し）とそのサブタスクは、まさに以下で説明する構造そのものです。

## タスク構造を構築する

タスクのフラットなリストだけでは、それらがどう関連しているかがわかりません。あるタスクを別のタスクの下にインデントすることで、ツリー構造（WBS — Work Breakdown Structure）を構築できます。親タスクはそのとき自動的に **サマリータスク** になります。

1. 構造の中でより深い位置に配置したいタスクを選択します。
2. **Alt+→** を押してインデントします。同じ操作にはもう1つショートカットがあり、**Alt+Shift+→** です — キーボード配列ですでに Alt+→ が別の用途に割り当てられている場合に便利です。どちらもまったく同じ動作をします。
3. マウス操作を好む場合は、タスクを右クリックしてコンテキストメニューから **インデント** を選びます。
4. 1段階行き過ぎてしまった場合は、**Alt+←**（または右クリック → **インデント解除**）でタスクを1段階戻せます。
5. まったく新しいサブタスクにはもっと速い方法があります。親タスクを右クリックして **サブタスクを追加** を選びます。これにより、先にタスクを追加してから別途インデントするのではなく、1ステップですでにインデントされた新しいタスクが作成されます。

タスクに少なくとも1つのサブタスクができると、そのタスクは自動的にサマリータスクになります。ガントチャート上のそのバーは、配下のすべてのサブタスクの最も早い開始から最も遅い終了までの全期間にまたがるようになり、自身の期間や日付を個別に設定することはできなくなります。そのためサマリータスクは常に派生値であり、直接入力するスケジュールではありません — サブタスクを削除したり移動したりすると、サマリータスクのバーは自動的に調整されます。

### 再インデントせずにタスクを移動する

タスクのレベルを変更する（インデント／インデント解除）以外に、構造そのものは変えずに、同じレベル内でタスクの位置を入れ替えることもできます。

- **Alt+↑** は、選択したタスクを、現在その上にあるタスクより上に移動します。
- **Alt+↓** はタスクを下に移動します。

これはツリーのどのレベルでも機能します。フェーズタスクを移動すると、そのすべてのサブタスクが自動的に一緒に移動します。

## マイルストーンの種類

マイルストーンは期間のないタスクで、開始・引き渡し・検査といった瞬間を示します。Open Planner Studio には、マイルストーンを追加する3つの方法があり、いずれも **タスク** リボングループの **マイルストーン** ボタンの横の矢印から行います。

- **開始マイルストーン** — フェーズまたはプロジェクトの開始を示します。
- **完了マイルストーン** — 引き渡しなど、完了を示します。
- **検査ポイント（必須）** — 実質的には、**必須（契約上）** フラグが最初からチェックされ、種別が直接 **検査** に設定された完了マイルストーンです。そのため検査の瞬間は、最初から契約上必須であることと検査であることの両方が一目でわかります。

ショートカット **Ctrl+M** の方が使いやすい場合は、汎用的なマイルストーン（「新しいマイルストーン」）が作成され、後から自分で名前と種別を設定できます。

マイルストーンを選択すると、プロパティパネルでも同じ区分が見られます。**マイルストーン** チェックボックスがオンになると、**マイルストーンの種類** フィールドで **自動**、**開始マイルストーン**、**完了マイルストーン** のいずれかを選べます。「自動」では、そのマイルストーンの関係に基づいてスケジューリングエンジンが振る舞いを判断します — マイルストーンに明確な開始的または完了的な性格がない場合はこれを選んでください。これとは別に **必須（契約上）** チェックボックスがあります。これは、開始マイルストーンか完了マイルストーンかにかかわらず、そのマイルストーンが契約上拘束力を持つことを示すフラグです。これにより、例えば開始マイルストーンを必須にすることもできますし、**検査ポイント** のように、必須の完了マイルストーンをワンクリックで設定することもできます。

## コードとフィールド：アクティビティコードとカスタムフィールド

大規模なスケジュールでは、WBS には収まらない追加の分類軸がすぐに必要になります。どの区画か、どの工種か、どの施工業者かなどです。これが **アクティビティコード** と **カスタムフィールド** の役割で、どちらも **コードとフィールド** ウィンドウ（**計画** タブの **構造** リボングループ、ボタン名は **コードとフィールド**）から管理します。

- **アクティビティコード** は自由に定義できる分類軸（例えば「場所」や「工種」）で、値のリストを持ちます — 各値には **コード**、**説明**、**色** があります。1つのタスクにつき、コードタイプごとに設定できる値は最大1つです。新しい分類軸を始めるには **コードタイプを追加** を、値を追加していくには **値を追加** を使います。
- **カスタムフィールド** は独自に型付けされたフィールドで、**テキスト**、**数値**、**整数**、**コスト**、**日付**、**はい/いいえ** のいずれかです。タスクテーブルに列として表示され、タスクごとに入力できます。「施工業者」（テキスト）や「許可取得済み」（はい/いいえ）といったフィールドを思い浮かべてください。

作成後は、タスクテーブルの列（必要であれば先に **表示 → 列…** で表示させます）またはタスクのプロパティパネルから、アクティビティコードを割り当てたりカスタムフィールドを入力したりできます。

### コードとフィールドでグループ化する

アクティビティコードとカスタムフィールドは、それらでグループ化して初めて真価を発揮します。リボンタブ **表示** に移動し、**グループ化** を開いて、**フィールド** の下でクラスタリングしたいアクティビティコードまたはカスタムフィールドを選びます。すると、タスクテーブルは WBS ツリーの代わりにグループヘッダーを表示します — 例えば、フェーズをまたいで区画ごと、または工種ごとにすべてのタスクをまとめて見るのに便利です。グループ化レベルは一度に最大2つまで設定できます（例えば最初に区画で、次に工種で）。

## メモ：タスクごとのチェックリスト

すべてのタスクには、プロパティパネルに **メモ** セクションがあります — 本質的には、そのタスクに付随する小さなチェックリストです。これは、スケジュール上の日付には収まらない、ちょっとしたアクション項目のためのものです。「まだ施工業者に確認が必要」「まだ資材を発注する必要がある」「図面 v2 を待っている」といったものです。

1. **+ メモを追加** をクリックします。テキストフィールドにフォーカスが当たった、空の新しい行が現れます。
2. メモの本文を入力します。
3. 項目が対応済みになったらチェックボックスをオンにします — テキストには取り消し線が引かれますが、メモ自体は表示され続けます（削除ではなく完了とマークされます）。そのため、タスクの履歴を後から読み返すことができます。
4. ゴミ箱アイコンを使うと、メモを完全に削除できます。

メモは純粋に情報提供のためのものです。スケジュールや計算には影響しないため、日付や期間として表現できない備考にはこれが適切な手段です。実際の運用例として、開いているメモと完了したメモが混在する様子を、中規模サンプル「Nieuwbouw 6 Rijwoningen De Akkers」（**ファイル → サンプル** のタグ *aantekeningen*／メモ）で確認できます。
