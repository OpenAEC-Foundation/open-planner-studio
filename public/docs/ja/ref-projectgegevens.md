# プロジェクト情報

**プロジェクト情報** ウィンドウには、プロジェクトのメタデータに加えて、スケジューリングオプションを含む **計算** セクションがあります。同じフォームは、**新規** のプロジェクトウィザードとしても機能します。

## 開く

- **設定**（リボンタブ）→ **プロジェクト** リボングループ → **プロジェクト情報**。
- 設定ウィンドウ（歯車 ⚙）→ **一般** タブ → **プロジェクト情報...**。
- **ファイル** → **プロジェクト情報** — Backstage 内の簡略版で、メタデータフィールドのみです（計算セクションはありません）。

**適用** はすべての変更を一度に確定します。**キャンセル**、**Esc**、またはウィンドウの外側のクリックで破棄します。**Enter** は適用と同じ動作をします。

## メタデータ

- **プロジェクト名** — タイトルバーとドキュメントタブに表示される名前。
- **説明** — 自由記述のテキスト。
- **担当者** と **会社** — 自由記述のテキスト。IFC ファイルに保存されます。
- **開始日** — 計算の起点となるプロジェクトの開始日。
- **終了日** — プロジェクトの参考用の終了日。

## 計算

このプロジェクトのスケジューリングオプションです — アプリではなくファイルに保存されるため、他のマシンでも一緒に持ち運ばれます。ここで何かを変更すると、**適用** の後にスケジュールが自動的に再計算されます。

- **クリティカルの定義** — **トータルフロート ≤ しきい値**（**しきい値（稼働日）** 付き、既定0）または **最長パス**。
- **フロート計算** — **最小（開始/終了）**（既定）、**開始フロート**、**終了フロート**。
- **オープンエンドタスクをクリティカルにする** — 後続タスクを持たないタスクをクリティカルとしてマークします。
- **準クリティカルとしてマーク** — チェックすると、追加の **しきい値**（既定2稼働日。単位は期間の表示設定に従うため、時間になる場合もあります）が表示されます。フロートの少ないタスクに「準クリティカル」のマーカーが付きます。
- **複数フロートパス** — チェックすると **方式**（**フリーフロート（ピーリング）** または **トータルフロート（ランキング）**）と **最大パス数**（既定10）が表示されます。計算は最も重要なフロートパスに番号を付けます。
- **ラグカレンダー** — 関係のラグをどのカレンダーで数えるか：**先行**（既定）、**後続**、**24時間**、**プロジェクトカレンダー**。

これらの結果の読み取り方は [クリティカルパスと高度な分析](docs://gids-kritiek-pad-analyse) で扱います。

## プロジェクトウィザード（新規）

**新規** は、同じウィンドウをウィザードとして開きます（タイトルは **新規プロジェクト**、ボタンは **作成**）。メタデータフィールドに加えて、ウィザードには次の項目があります。

- **フェーズテンプレート** — **空**、**住宅建築**、**非住宅建築 / 改修**：新しいプロジェクトにフェーズ構造を用意します。
- **シフト** — 時間計画が有効な場合のみ表示されます：**日勤**（既定）、**2交代**、**3交代**、**24/7**。
- **祝日セット** — プロジェクトカレンダーを生成します：国を選ぶ（該当する場合は地域と建設業休暇付き）、**祝日なし**、または **カスタム…**。カスタムを選ぶと、作成直後にカレンダーダイアログが開き、カレンダーを手作業で構成できます。[カレンダーダイアログ](docs://ref-kalenderdialoog) を参照してください。

計算セクションはウィザードにはありません。作成後、上記のいずれかの入口から設定してください。
