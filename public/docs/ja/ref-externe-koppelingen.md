# 外部リンク

**外部（プロジェクト間）リンク** ウィンドウは、このプロジェクト内のタスクと、別のプロジェクトファイル内のタスクとの間の依存関係を記録します — 例えば、あなたの着工前に終わる必要がある外構工事プロジェクトなどです。

## 開く

**関係** タブ → **外部リンク…** ボタン。タスクをちょうど1つ選択している必要があります。そうでない場合は「外部リンクを追加するには、タスクを1つだけ選択してください。」と表示されます。

## 凍結されたアンカー

外部リンクは、ソースプロジェクトに対してライブに計算されるわけではありません。追加すると、ソースタスクの関連する日付（方向と関係タイプに応じて開始または終了）が固定の **アンカー日付** として保存されます。計算はその日付を境界として使います。その後ソースプロジェクトが変更されても、リンクを **更新** するまで何もずれません。

## 2つの方法

- **ソースファイル** — **最近使用したファイルを選択** の下でファイルを選びます。これは読み取り専用で読み込まれます（「ソースファイルは読み取り専用で読み込まれます — ドキュメントとして開かれることはありません。」）。次にリストから **ソースタスク** を選びます。アンカー日付はそのタスクから自動的に読み取られ、下部に表示されます。この方法にはデスクトップアプリと、少なくとも1つの最近使用したファイルが必要です。
- **手動（フォールバック）** — 手元にファイルがない場合（またはブラウザ版の場合）：外部タスクの **プロジェクトID** と **タスクID** を貼り付け、任意で **タスク名** を入力し、**アンカー日付** を自分で入力します。手動リンクは、実際に更新でソースが見つかるまで「古い」とマークされます。

## 共通フィールド

- **方向** — **先行（外部 → 自分）**：外部タスクが自分のタスクを駆動する。または **後続（自分 → 外部）**：自分のタスクが外部タスクを駆動する。
- **関係タイプ** — FS、SS、FF、SF。
- **ラグ（稼働日）** — アンカーに加算される待機時間（負の値なら重なり）。

**リンクを追加** はリンクを保存します（必須フィールドが入力されるまで無効です）。**キャンセル** は追加せずに閉じます。

## 管理、更新、ソースが見つからない場合

既存のリンクは、関係パネルの **外部リンク** の下に一覧表示されます。

- リンクごとに：ソースタスク、タイプ、アンカー、そしてソースが（もはや）読み込めなかった場合に表示される **古い** バッジ — 「ソースが読み込まれていません — 更新するには再インポートしてください」という説明付きです。
- **このリンクを更新** — この1つのリンクのソースファイルを再読み込みし、アンカーを更新します。
- **外部アンカーを更新** — 参照されているすべてのソースファイルを再読み込みし、すべてのアンカーと古さの状態を更新します。その後、いくつのアンカーが更新され、いくつが古いままだったかを報告するステータス行が表示されます。
- **削除** — リンクを削除します。
- 更新はファイルを読み込むため、デスクトップアプリでのみ機能します。ブラウザ版では「ソースファイルの読み込みはデスクトップアプリでのみ可能です。手動フォールバックを使用してください。」と表示されます。
