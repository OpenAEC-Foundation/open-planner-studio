# レポートと印刷

スケジュールは、共有できて初めて完成したと言えます — 現場会議のための紙、プレゼンテーションの画像、あるいは今後の予定とすでにずれたことの概要としてです。それが **レポート** タブの役割で、3つのレポートの種類と印刷プレビューがあります。

## ここで学ぶこと

- **レポート** タブにある3つのレポートの種類：ガント印刷、マイルストーン一覧、差異。
- 印刷プレビューの動作：用紙サイズ、向き、オン／オフを切り替えられる要素。
- レポートを実際に印刷する方法、またはファイルとして保存する方法。
- このアプリで **Ctrl+P** が何をするか。

## レポート画面に入る

同じ画面に入る方法は3つあります。**レポート** リボンタブをクリックする、**Backstage → 印刷**（レポート画面を直接開きます）に移動する、または **Ctrl+P** を押す、です。3つとも同じ場所に着地します — 独立した「印刷」ダイアログはありません。レポート画面 *そのもの* が印刷プレビューです。

画面は2列に分かれています。左側には上部に **レポートの種類** の選択があるオプションパネル、右側には左側で設定を変更するたびに即座に更新されるライブプレビューがあります。

## 3つのレポートの種類

### ガント印刷

ガントバーの完全な整形済みプリントアウトです — オプションブロックを持つ唯一のレポートの種類です。

- **用紙**：A4、A3、A1。
- **向き**：横向きまたは縦向き。
- **用紙に自動調整**（オン＝選んだサイズにスケジュールが自動的にスケーリングされる）、またはオフにした場合の手動 **ズーム** スライダー。
- **バー上のタスク名**、**進捗を表示**、**クリティカルパス**、**フロートを表示**、**依存関係**、**週末**、**凡例** のトグル。
- **会社** フィールド（プロジェクト設定から自動入力されますが、ここで個別に編集も可能です）と、（プロジェクト情報から取得される、読み取り専用の）**作成者**。

その上のサマリーブロックには、プロジェクト内のタスク数、末端タスク数、クリティカルタスク数、関係数がライブで表示されます。

### マイルストーン一覧

プロジェクト内のすべてのマイルストーンの表です。WBS、名前、種類（自動／開始／完了）、日付、根底にある制約または期限、余裕、必須かどうか、状態（予定どおり／クリティカル／遅延）です。サマリーブロックには、マイルストーンの合計数、必須の件数、遅延の件数が表示されます。このレポートには用紙サイズ／向きの設定はありません — 表示されているとおりに表を印刷します。

### 差異

現在のスケジュールをアクティブなベースラインと比較します。ベースライン開始／終了と現在の開始／終了、開始・終了の稼働日数の差、そしてタスクごとの状態（予定どおり／遅延／前倒し／新規／削除）です。アクティブなベースラインがない場合、画面は空のレポートを表示する代わりに、その旨を明示します。サマリーブロックには、変化があれば、プロジェクトの終了日の稼働日数でのずれも表示されます。このレポートが何か役立つ情報を教えてくれるようになるには、先にベースラインを記録する必要があります — その方法についてはガイド [ベースラインと進捗](docs://gids-baselines-voortgang) を参照してください。

## 印刷とエクスポート

オプションパネルの下部には常に **印刷...** ボタンがあります — レポートを含む別の印刷ウィンドウを開き、すぐにブラウザ／OS の印刷ダイアログを起動します。ガントレポートの場合、そのウィンドウは選んだ用紙サイズと向きを使います。マイルストーンレポートと差異レポートは、表示されているとおりの表を印刷します。

**PDF エクスポート** ボタンがあるのはガントレポートだけです。これは現在のプレビューを実際の PDF ファイル（ファイル名は `-planning.pdf` で終わります）として保存します — 選んだ用紙サイズと向きの物理的な寸法に合わせた1ページです。この PDF ファイルは **ベクター形式** です。バー、線、テキストは、単一の埋め込み画像ではなく PDF の描画命令として保存されるため、どのズームレベルでも鮮明なままで、テキストはどの PDF ビューアでも選択・検索できます。これはラテン文字、キリル文字、ギリシャ文字のテキストに当てはまります。プロジェクトに中国語、日本語、韓国語、アラビア語、ペルシャ語のテキストが含まれる場合、エクスポートはそのテキストについて自動的にラスター画像にフォールバックします — 表示は引き続き正しく行われますが、選択や検索はできません。システムの印刷ダイアログを介さずにメールで送ったりアーカイブしたりするのに便利です。直接印刷したい場合（または、上で設定したのとは別の用紙サイズを選ぶためにシステムダイアログ経由で PDF に保存したい場合）は、**印刷...** を使ってください。

## 実践でのレポート

各レポートの種類は、それぞれ異なる会話のためのものです。

- **ガントレポート** は昔ながらの現場会議向けの配布資料です。クリティカルパスがハイライトされ、非クリティカルなバーにはフロートが見え、凡例が各色の意味を説明します。聴衆がスケジュールをまだよく知らない場合は **バー上のタスク名** と **進捗を表示** をオンに、別途タスクリストを配布する場合は A1 で見やすくするためにオフにしてください。
- **マイルストーン一覧** は、何十行ものタスクをページめくりしなくても、重要な日付だけを知りたい人向けです — 例えば、必須の引き渡し日が守られているかどうかを主に知りたい施主などです。表内でマイルストーン名の前にある ◆ 記号は **必須** マイルストーンを示します。
- **差異レポート** は、軌道修正についての会話です。ベースラインに対してどのタスクがどれだけの稼働日数遅れているか。このレポートの実例は、ショーケース [Nieuwbouw Appartementencomplex De Vaart](examples://showcase-appartementencomplex.ifc) で確認できます。2つのベースライン（契約ベースラインと変更指示後の再ベースライン）があり、それぞれ独自の進捗と状態日を持っています — ベースラインと現在のスケジュールの間に実際の差異が生じたとき、Δ 列がどう埋まっていくかを見るのに良い例です。

右側のライブプレビューは、左側の設定を変更するたびに更新されます — 独立した「更新」ボタンはなく、印刷時にのみ計算されるものは何もありません。

## さらに詳しく

- 差異レポートは、ベースラインが記録されるまで比較対象がありません — ガイド [ベースラインと進捗](docs://gids-baselines-voortgang) を読んでください。
- ガントレポートに表示されるクリティカルパスとフロートは、ガントビュー自体と同じ計算から来ています — その読み取り方については、ガイド [クリティカルパスと高度な分析](docs://gids-kritiek-pad-analyse) を読んでください。
