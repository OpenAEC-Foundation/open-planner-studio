# 山崩しオプション

**リソースの山崩し** ウィンドウは、タスクをずらすことで超過割り当てを解決します。動作は2段階です。**計算** で提案を作成し（まだ何も変わりません）、**適用** で実行します。

## 開く

**リソース** → **山崩し** リボングループ → **山崩し…**。**Esc**、閉じるボタン、またはウィンドウの外側のクリックで、適用せずに閉じます。

## オプション

- **フロート内でのみ山崩しする（平滑化）— プロジェクト終了日は変わりません** — チェックすると、山崩しは各タスクのトータルフロート内でのみタスクをずらします。終了日は動きませんが、すべての競合を解決できるとは限りません。チェックなし（既定）の場合、すべての競合を解決するためにプロジェクト終了日が延びることがあります。
- **リソース** — リソースごとのチェックボックス：どのリソースが参加するか。資材リソースはここには表示されません（資材は山崩しされません）。既定ではすべてのリソースがオンです。

## 計算

最新の計算結果が必要です。そうでない場合、ウィンドウには「山崩しの前にスケジュールを計算してください（F5）。」と表示されます。リソースが1つもチェックされていない間もボタンは無効です。オプションを変更すると、それ以前の提案は無効になります — 再度計算してください。

## 提案（プレビュー）

- **プロジェクト終了日の行** — 「変更なし（日付）」、またはプロジェクトが延びる場合は「旧日付 → 新日付」（赤）。
- **表** — ずれるタスクごとに：**タスク**、**旧開始日**、**新開始日**、**移動日数**。ロジックを通じて一緒にずれる、リソースを持たない後続タスクも含まれます。
- 何もする必要がない場合、ウィンドウは「移動が必要なタスクはありません — スケジュールはすでに競合がありません。」と報告します。

## 残っている競合

ルールに収まらないタスクについて、タスクごとに競合日数と理由が表示されます。

- 「…はピーク時に…単位/日必要ですが、キャパシティは…です — 移動では解決できません。」— ある割り当てがピーク時にリソースの容量を超える量を必要としています。単位/日を下げるか、最大単位数を上げてください。
- 「このリソースはこのタスクが必要とするすべての日に稼働していません — 移動では解決できません。」— タスクとリソースのカレンダーの不一致です。
- 「フロート内の空きキャパシティが不足しているため、この競合を解決できません。」— 主に平滑化を使用時：利用可能なフロート内に空き枠がありません。

## 適用と取り消し

**適用** は提案を実行し、ウィンドウを閉じます。**キャンセル** は変更せずに閉じます。適用した山崩しを取り消すには、**山崩しを解除**（同じリボングループ）または Ctrl+Z を使ってください。
