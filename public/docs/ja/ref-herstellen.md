# クラッシュ後の復元

デスクトップアプリは、作業内容の復元用スナップショットを自動的に保持します。アプリが予期せず終了した場合（クラッシュ、停電）、次回の起動時にその作業を復元するかどうかを尋ねます。

## 自動保存の仕組み

- 変更のたびに（1秒未満で）、アプリは開いているドキュメントごとのスナップショットを、独自のデータフォルダに書き込みます — 一度も保存されていないドキュメントを含む、開いているすべてのタブが対象です。
- これは保存の代わりにはなりません。プロジェクトファイル自体は変わりません。そのため、引き続き Ctrl+S で作業を保存してください。
- スナップショットは、復元ウィンドウで選択（**復元** または **復元しない**）を行った時点で削除されます。
- **デスクトップアプリのみ。** ブラウザ版には自動保存も復元もありません — ブラウザでは自分でこまめに保存してください。

## 「未保存の作業を復元」ウィンドウ

スナップショットが見つかると、起動時に表示されます：「Open Planner Studio が正常に終了しませんでした。以下のドキュメントには復元可能な未保存の変更があります：」。ドキュメントごとに次の項目が表示されます。

- **名前**（ファイル名またはプロジェクト名。名前がない場合は「無題のプロジェクト」）。
- ドキュメントが一度でも保存されたことがある場合の **ファイルパス**。
- スナップショット内の **タスク数**。
- **保存日時** — 最新のスナップショットの時刻。

## 選択肢

- **復元**（または **Enter**）— 一覧表示されたすべてのドキュメントが、開いているタブとして戻ってきます。それらは未保存として扱われるため、自分で保存してください。
- **復元しない** — スナップショットは破棄され、空のプロジェクトから始めます。
- **閉じるボタン**、**Esc**、またはウィンドウの外側のクリック — 安全に先延ばしにします。何も破棄されず、何も復元されません。次回の起動時に、この質問が再び表示されます。
