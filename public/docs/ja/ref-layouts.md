# レイアウトの保存と読み込み

レイアウトとは、保存されたビュー設定です。列、グループ化、並べ替え、フィルター、タイムスケールを1つにまとめたものです。レイアウトはアプリ全体（この端末上）で共有され、1つのプロジェクトファイルに属するものではないため、どのドキュメントでも使用できます。

## 開く

**表示** → **レイアウト** リボングループ。自分のレイアウトを選ぶピッカーと、3つのボタンがあります。

- **名前を付けて保存…** と **管理…** — どちらも（下記の）**レイアウトの管理** ウィンドウを開きます。
- **更新** — ピッカーで選んだレイアウトを現在のビューで上書きします。**(なし)** が選ばれている間は無効です。

ピッカーでレイアウトを選ぶと、即座に適用されます。

## レイアウトの管理ウィンドウ

保存されたレイアウトがない場合、ウィンドウには「保存されたレイアウトはまだありません。」と表示されます。それ以外の場合は、レイアウトごとに1行、次の項目があります。

- **名前** — 行内で直接編集できます（名前の変更）。
- **適用**（チェックマーク）— 先に確認を求めます：「レイアウト「…」を適用しますか? 現在の列/グループ化/並べ替え/フィルター/スケールが置き換えられます。」
- **更新** — 確認なしで、現在のビューでレイアウトを上書きします。
- **削除**（ゴミ箱アイコン）— 先に確認を求めます。

確認はアプリ内の小さなダイアログとして表示されます。**Esc** または **キャンセル** で中止できます。

## レイアウトを名前を付けて保存…

ウィンドウの下部：**名前** を入力して **保存** をクリックすると、現在のビューが新しいレイアウトとして保存され、アクティブなレイアウトになります。名前を入力しない場合、レイアウトには既定の名前「名前」が付きます。

## レイアウトが記録する内容

- 列（表示、順序、幅）— [列の選択](docs://ref-kolommen) を参照。
- グループ化と並べ替え（**表示** → **グループ化…** / **並べ替え…**）。
- フィルター — [フィルター](docs://ref-filters) を参照。
- ガントのタイムスケール。

含まれないもの：ズームレベルの詳細、パネルの幅、選択状態。
