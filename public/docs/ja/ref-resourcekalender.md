# リソースカレンダー

**リソースカレンダー** ウィンドウでは、1つのリソース独自のカレンダーを編集します — 例えば週4日勤務の班などです。フォームは [カレンダーダイアログ](docs://ref-kalenderdialoog) と同一です。この記事では違いだけを説明します。

## 開く

- リソースパネルを開く：**リソース** → **管理** リボングループ → **リソース**（フルパネル）または **ドッキング**（ガントの横にドッキング）。
- リソースの **カレンダー** 列で、カレンダーを選び、その横の鉛筆アイコン（**編集…**）をクリックして編集します。同じドロップダウンから新しいカレンダーも作成できます。

## カレンダーダイアログとの違い

- **一度に1つのカレンダーのみ** — 左側のライブラリリストやプロジェクト既定の星印はなく、フォームだけです。
- **適用** でカレンダーを保存します。**キャンセル**、**Esc**、閉じるボタン、またはウィンドウの外側のクリックは変更を破棄します。
- **自動再計算はありません** — リソースカレンダーは CPM の日付を変更しません。負荷（ヒストグラム）と山崩しに影響しますが、それらはそれぞれ F5 または **山崩し…** で自分で再実行する必要があります。

## フィールド

完全なフィールドリファレンスについては [カレンダーダイアログ](docs://ref-kalenderdialoog) を参照してください：**名前**、**稼働日**（月〜金と常時（24/7）のプリセット付き）、**開始（時）** / **終了（時）** / **1日の時間数**、（時間計画がオンのときの）**稼働時間** セクション、**祝日を生成…**、**祝日** リスト。
