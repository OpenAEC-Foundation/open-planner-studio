# ベースラインと進捗

更新されないスケジュールは単なる予測にすぎません。作業が始まったら、2つのことを同時に見たいはずです。当初何が合意されたのか、そして今実際に何が起きているのかです。**ベースライン** は前者を凍結し、**進捗** と **状態日** は後者を追跡します。このガイドでは、ベースラインの保存と管理の方法、差異を可視化する方法、進捗の入力方法、そして状態日がスケジュールに正確に何をするのかを説明します。

## ここで学ぶこと

- ベースラインの保存と管理、そしてどのベースラインがアクティブかについて。
- 差異を確認する方法：ガントのベースラインオーバーレイと差異レポート。
- 進捗の入力 — パーセンテージ、実績日 — パネル、タスクダイアログ、コンテキストメニューから。
- 状態日：未着手のタスクとマークされていないマイルストーンに何をするか。
- 順序違反の警告：それが何を意味し、どう解決するか。
- 進捗ラインの読み取り方。

[Nieuwbouw 6 Rijwoningen De Akkers](examples://showcase-rijwoningen-de-akkers.ifc)（着工前のベースライン1つ、加えてプロジェクト途中の進捗と状態日）と、[Nieuwbouw Appartementencomplex De Vaart](examples://showcase-appartementencomplex.ifc)（2つのベースライン — 契約ベースラインと変更指示後の再ベースライン — それぞれ独自の進捗と状態日を持つ）を見ながら進めてください。

## ベースラインの保存と管理

**計画** タブの **ベースラインと進捗** リボングループから **ベースライン** ウィンドウを開きます。**ベースラインを保存…** は、提案された名前（「ベースライン1 — [日付]」）で新しいベースラインを即座に保存し、**ベースラインを管理…** は同じウィンドウを開いて確認・名前変更・削除を行います。

このウィンドウには、保存されたすべてのベースラインの表が表示されます。**アクティブ** のラジオボタン、（直接編集できる）**名前**、**作成日**、削除ボタンです。一度にアクティブにできるベースラインはちょうど1つです — それがガントのオーバーレイと差異レポートの比較対象になります。アクティブなベースラインを削除すると確認が求められます（削除後は、別のベースラインを選ぶか新しいベースラインを保存するまで、どのベースラインもアクティブになりません）。最後の計算以降スケジュールが古くなっている場合、ウィンドウには「新しいベースラインを保存」の横に先に再計算するようヒントが表示されます — 古いスケジュールに対して保存したベースラインは、間違った日付を凍結してしまうことになるためです。

ベースラインはスナップショットです。保存した瞬間の、すべてのタスクの開始日、終了日、（マイルストーンの場合は）日付です。その後さらにスケジュールを変更しても、自分で新しいベースラインを保存するまで、ベースラインは変わりません。

## 差異を確認する

### ガント内：ベースラインオーバーレイ

**表示 → ベースラインと進捗リボングループ → ベースラインオーバーレイ** でオーバーレイをオンにします。すべてのタスクバーの下に、ベースラインの色で、元のベースラインの日付における細いサブバー（マイルストーンの場合はダイヤモンド）が表示されます。メインのバーがサブバーを越えて伸びていれば、別のレポートを開かなくても、タスクがベースラインに対してどれだけ遅れているかが一目でわかります。

### レポートとして：差異レポート

**レポート** タブに移動し、**レポートの種類** で **差異** を選びます。レポートには、タスクごとに **ベースライン開始**、**ベースライン終了**、**現在の開始**、**現在の終了**、**Δ 開始（稼働日）**、**Δ 終了（稼働日）**、そして **状態**（**予定どおり**、**遅延**、**前倒し**、ベースライン以降に追加されたタスクの場合は **新規**、それ以降に削除されたタスクの場合は **削除**）が表示されます。上部には、レポートがタスク数の合計、遅延しているタスク数と前倒しになっているタスク数、そして — プロジェクト終了日がずれている場合は — ベースラインに対する稼働日数の差を示す行があります。アクティブなベースラインがない場合、レポートは空のテーブルを表示する代わりに、その旨を明示的に伝えます。

## 進捗を入力する

進捗は3か所で設定でき、いずれも効果は同じです。

1. **プロパティパネル** — 選択したタスクの下の **進捗** セクション：**進捗（%）** のスライダー、そして（通常のタスクの場合）**実績開始**／**実績終了** フィールド、または（マイルストーンの場合）単一の **実績日** フィールド。実績開始日を入力せずにパーセンテージを0%より大きくすると、計画上の最早開始日で自動的に入力されます。100%を下回るまで戻すと、入力していた実績終了日も再びクリアされます。
2. **タスクダイアログ** — **タスクの編集** ウィンドウの、同じ **進捗** セクション。
3. **コンテキストメニュー** — タスクを右クリックし、**進捗** サブメニューから、固定ステップの **0%**、**25%**、**50%**、**75%**、**100%** を選びます。パネルを開かずにさっと更新したいときに便利です。中間のパーセンテージや特定の実績日を入力するには、パネルまたはタスクダイアログを使ってください。

実績日は状態日より後にすることは絶対にできません。それより後の日付を入力しようとすると、アプリはエラーで拒否します。これは意図的な境界です。「事実」（実際に起きたこと）は、定義上、進捗を記録している時点より未来にあってはならないからです。

## 状態日

**状態日**（計画タブの **ベースラインと進捗** リボングループ、**状態日** フィールド）は、スケジュール内での「今日」— 進捗をその時点として記録した瞬間を示します。設定すると、同時に2つのことが起きます。

- まだ開始していないタスクやマイルストーン（進捗0%、実績開始なし）は、ロジック（先行タスク、関係）がそれより早い開始を許していたとしても、状態日より早く開始することはできません。計算された最早開始日は状態日で「底上げ」されます。
- すでに開始または完了しているタスクは、実績日をそのまま保持します — それらは状態日によって上書きされることは決してありません。

これは中規模のショーケースで正確に確認できます。状態日を2027年5月20日に設定すると、まだ開始していない複数のタスク（例えば異なる住宅の煉瓦積みや配管工事）の最早開始日が、まさにその日付に固定されます。それらは異なる住宅で動いており、状態日による底上げがなければさまざまな、より早い日付で開始していたはずであってもです。

### マークされていないマイルストーンが「右にずれる」理由

計算上、マイルストーンは期間ゼロのタスクにすぎないため、同じルールが適用されます。まだ完了とマークされていない場合（100%でも実績日でもない）、その計算された日付は状態日より前になることはできません。マイルストーンを完了とマークしないまま状態日を先に進め続けると、ガント上に表示される日付もそれに合わせて右にずれ続けます。根底にあるタスクは何も変わっていないにもかかわらずです — スケジュールは実質的に「まだチェックを入れていないなら、この瞬間は過去にあってはならない」と言っているのです。マイルストーンを実績日付きで完了とマークすると、その固定された日付にスナップして戻り、ずれなくなります。

## 順序違反の警告

状態日が設定されると、計算は記録された事実（実績開始日／実績終了日）が関係のロジックと矛盾していないかどうかもチェックします — 例えば、スケジュール上ではまだ終わっていないはずの先行タスクがあるのに、すでに開始している後続タスクなどです。このようなケースは **順序違反**（out-of-sequence）と呼ばれ、画面下部のステータスバーに警告として表示されます（「N 件の順序違反の関係」）。件数はツールチップで確認できます。これは警告であり、計算を止めるエラーではありません — 計算はそのまま続行されます。

順序違反の警告は、実際の状況を正確に記録することで解決します。関係するタスクに、欠けている、または誤っている実績開始日／実績終了日を入力してください（上記のとおり、パネル、タスクダイアログ、コンテキストメニューから）。そうすることで、記録された事実が、論理的に先行していなければならなかった内容と再び整合します。多くの場合、これは単に、実際にはすでに終わっているのにスケジュール上ではまだそうマークされていないタスクがある、というだけのことです。

## 進捗ライン

**表示 → ベースラインと進捗リボングループ → 進捗ライン** で進捗ラインをオンにします。オレンジの破線（4/4ダッシュ、状態日ラインと同じスタイル）が描かれ、すべてのタスクについて、その進捗率に対応する位置に点をプロットし、それを状態日に結びます — 古典的なジグザグパターンです。状態日の左側への折れ曲がりは、タスクが経過時間から予想される進捗より遅れていることを意味し、右側への折れ曲がりは進んでいることを意味します。進捗ラインはジグザグの背骨として状態日の垂直線自体をすでに描いているため、独立した **状態日ライン** のトグル（同じリボングループ）は進捗ラインがオンの間は後退します — 進捗ラインをオフにして、それでも状態日を単純な垂直線として表示したい場合にのみ、再び表示されます。
