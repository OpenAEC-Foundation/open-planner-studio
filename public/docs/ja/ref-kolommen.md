# 列の選択

**列** ウィンドウは、テーブル タブがどの列を、どの順序で、どれくらいの幅で表示するかを制御します。（ガントの左側にあるタスクテーブルは、WBS、タスク名、期間という固定列を持ちます。）

## 開く

**表示** → **表示** リボングループ → **列…**。すべての変更は即座に適用されます — 独立した OK ステップはありません。**閉じる**、**Esc**、閉じるボタン、またはウィンドウの外側のクリックで閉じます。

## 選択された列

列ごとに1行、次の項目があります。

- **ドラッグハンドル** — 行をドラッグして列の順序を変更します。
- **表示** — チェックを外すと、リストから削除せずに列を非表示にします。
- **名前** — テーブルに表示されるフィールドのラベル。
- **幅** — ピクセル単位（最小40）。

## 利用可能なフィールド

選択された列の下に **利用可能なフィールド** のリストがあります。まだ列になっていないすべてのフィールドです。クリックすると列として追加されます。標準フィールドのほかに、分析フィールドの **マイルストーン**、**フリーフロート**、**干渉フロート**、**準クリティカル**、**フロートパス**、加えて **リソース** とプロジェクトのアクティビティコード・カスタムフィールドがあります。3つのフロートフィールドとフロートパスは、対応するスケジューリングオプションでの計算後にのみ値を持ちます — [クリティカルパスと高度な分析](docs://gids-kritiek-pad-analyse) を参照してください。

## 既定値に戻す

**既定値に戻す** は標準の列セットを復元します。追加したフィールドは利用可能なフィールドのリストに戻ります。

列セットは保存されたレイアウトの一部です — [レイアウトの保存と読み込み](docs://ref-layouts) を参照してください。
