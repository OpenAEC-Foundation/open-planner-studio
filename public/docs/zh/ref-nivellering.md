# Nivelleringsopties

**资源平衡**窗口通过移动任务来解决超额分配问题。它分两步进行：**计算**生成方案（尚未发生任何改变），**应用**执行该方案。

## 打开方式

**资源** → **平衡**功能区组 → **平衡…**。**Esc**、关闭图标或点击窗口外部会关闭窗口而不应用。

## 选项

- **仅在浮动范围内平衡（平滑）— 项目结束日期保持不变** — 勾选时，平衡只会在任务的总浮动范围内移动任务：结束日期不能移动，但并非每个冲突都能因此解决。未勾选（默认）时，项目结束日期可以延长以解决所有冲突。
- **资源** — 每个资源一个复选框：哪些资源参与。材料资源不会出现在这里（材料不参与平衡）。默认所有资源都开启。

## 计算

需要一个最新的计算；否则窗口会显示“请先计算计划 (F5)，再进行平衡。”若没有勾选任何资源，按钮也会不可用。任何选项更改都会使先前的方案失效——需要重新计算。

## 方案（预览）

- **项目结束日期行** — “不变（日期）”，或若项目延长则显示“旧日期 → 新日期”（红色）。
- **表格** — 按已移动的任务：**任务**、**原开始**、**新开始**和**移动天数**。通过逻辑一同移动的非资源后续任务也包含在内。
- 如果没有需要处理的内容，窗口会报告“没有任务需要移动 — 计划已无冲突。”

## 剩余冲突

不符合规则的任务，按任务显示冲突天数和原因：

- “……峰值需求为 ……单位/天，而产能为 ……— 无法通过移动解决。” — 某个分配在峰值时需求超过资源产能；请降低单位/天或提高最大单位数。
- “该资源并非在此任务所需的所有天数中都工作 — 移动无法解决此问题。” — 任务与资源之间的日历不匹配。
- “浮动范围内的空闲产能不足，无法解决此冲突。” — 多见于平滑模式：可用浮动范围内没有空闲窗口。

## 应用与撤销

**应用**执行方案并关闭窗口；**取消**关闭窗口而不做更改。使用**清除平衡**（同一功能区组）或 Ctrl+Z 撤销已应用的平衡。
