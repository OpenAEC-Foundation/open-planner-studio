# Im-/export

Open Planner Studio 默认将项目存储为 IFC——不会有单独的项目文件伴随其旁。但有时计划也需要存在于应用之外：在 Primavera P6 中、在 Microsoft Project 中，或作为供电子表格使用的扁平表格。本指南解释原生 IFC 格式究竟意味着什么、每种导出格式携带和不携带哪些内容，以及导入/导出在应用中的位置。

## 您将在此学到什么

- “IFC 是原生格式”对打开和保存究竟意味着什么。
- 导出到 MS Project（MSPDI）和 Primavera P6 XML 时，哪些内容会随行，哪些不会。
- CSV 导出包含什么内容——以及哪些内容被刻意排除在外。
- 在哪里导入和导出：**Backstage → 导出**和**Backstage → 导入**。
- 扩展如何添加额外的导入格式。

## IFC：原生格式

一个 Open Planner Studio 项目*就是*一个 IFC 4x3 文件（buildingSMART 标准）。不存在单独伴随的 JSON 或项目文件：**保存**和**打开**（Backstage，或 **Ctrl+S**/**Ctrl+O**）直接读写 IFC。这意味着您在应用中所做的一切——任务、WBS、带限制的关系、资源和分配、日历（项目日历和资源日历）、基线、进度、备注、活动代码和自定义字段、项目之间的外部链接——都会保存到同一个文件中，并在您下次**打开**它时完整地恢复。如果您在应用中遇到一种新的项目数据类型，可以假定它能通过 IFC 往返；如果某些内容*不能*往返，下文会明确指出。

IFC 也是本应用连接 OpenAEC 工具套件其余部分的方式：同一个文件可以被 BIM 软件读取，用于 4D 关联（计划与建筑模型并列）。

## 导出为其他格式

打开 **Backstage → 导出**，有四种格式：

- **CSV（分号分隔）** — 通用表格导出。所有任务的日期和工期。
- **MS Project XML** — 在 Microsoft Project 中打开。完整的 WBS 结构。
- **Primavera P6 XML** — 适用于 Oracle Primavera P6。
- **IFC 4x3** — buildingSMART 标准，与原生格式相同（适合用作“另存为”单独文件，或分享一份副本而不影响您其他打开的文档）。

每种格式都有自己的限制：目标格式越丰富，随行的内容就越多，但三种外部格式都不是 IFC 的完整镜像。

### CSV

CSV 导出**仅包含任务表格**：WBS 编码、名称、工期（天）、开始、完成、前置任务（作为文本代码，例如 `2.1FS+3d`）、任务类型、状态、完成度（%）、实际开始/完成、关键（是/否）、总浮动和描述。**资源、分配、日历和基线被刻意排除在外**——CSV 纯粹是一份任务表格，供想在电子表格中查看或编辑计划的用户使用，而不是一次完整保真的项目交换。当您把 CSV 文件重新**导入**时，基线因此会保持为空（本来就没有可读取的内容）。

### MS Project XML（MSPDI）

MSPDI 比 CSV 丰富得多：资源、分配（包括其负荷曲线）、日历和基线都会随行。但并非一切都能在 MSPDI 中表达。导出时，应用会在开发者控制台中警告（`console.warn`）每一处丢失的内容，并给出受影响项目的确切数量：

- **项目之间的外部链接**会被丢弃（另一任务的“幽灵”引用仅保留在应用内部）。
- **软性必须开始于/必须完成于限制**（软性 `MSO`/`MFO`）会降级为 SNET/FNET——MSPDI 的代码 2/3 是*硬性*的（Must），因此软性版本的上界会丢失。硬性 `MSO`/`MFO` 会精确导出。
- **次要限制**会丢失——MSPDI 每个任务只有一个限制字段。
- **Hammock 任务**（派生工期）会作为带有计算日期的普通任务导出——MSPDI 没有原生的 hammock/LOE 类型。
- **任务备注**刻意**不**导出，尽管 MSPDI 有一个 `<Notes>` 字段：我们的备注是带复选框的清单形式，无法干净地转换为纯文本。
- **关键路径定义**（接近关键模式/阈值）和其他排程选项在 MSPDI 中无法原生表达，因此会丢失——这些只能通过 IFC 保留。

### Primavera P6 XML

与 MSPDI 类似的权衡取舍，外加几个 P6 特有的特殊情况：

- **项目之间的外部链接**和 **Hammock 任务**会以与 MSPDI 相同的方式被丢弃/简化，各自附带警告。
- **任务备注**在这里也会被省略——P6 XML 没有适合的字段容纳它们。
- 关系上的**百分比延时**（例如前置任务工期的 40%）会被“固化”为固定的天数，因为 P6 没有百分比延时的概念。
- **按日历计天数的延时**（以按日历计天数而非工作日表示的延时）会作为普通的按小时延时导出——P6 没有针对每个关系的独立延时单位。
- **LATE_PEAK** 负荷曲线在 P6 中没有对应项，会以最接近的近似值（“Early Peak”）导出。
- 排程选项（与 MSPDI 一样）不会被导出。

这些警告并非疏忽——而是一种刻意的、明确的选择：每丢失一项就有一条可见的警告，胜过悄无声息的数据丢失。打开例如展示项目 [Nieuwbouw 6 Rijwoningen De Akkers](examples://showcase-rijwoningen-de-akkers.ifc)（它有任务备注和一条带百分比延时的关系）并导出为 P6 或 MS Project XML：开发者控制台会随即确切显示哪些项目被丢弃或简化，以及数量。

## 导入

**文件 → 打开**（或 **Backstage → 打开**）接受 `.ifc`、`.csv` 和 `.xml` 文件。对于 `.xml` 文件，应用会根据内容自行检测它是 Primavera P6 文件还是 MS Project 文件。如上所述：CSV 或 P6 导入生成的项目**没有基线**（源文件中本来就没有），而 IFC 和 MSPDI 会随行携带基线。

## 扩展导入器

除了上述固定格式外，已安装的扩展可以添加自己的导入器——例如针对默认不支持的格式。它们会显示在 **Backstage → 导入**下，各自带有自己的名称、描述和匹配的文件扩展名；未安装任何导入扩展时，该部分为空。查看 **Backstage → 扩展**了解有哪些可用。

## 延伸阅读

- 基线只通过 IFC 和 MS Project XML 随行，而不通过 CSV 或 P6——阅读指南 [Baselines & voortgang](docs://gids-baselines-voortgang) 了解如何记录基线。
