# Projectinformatie

**项目信息**窗口保存项目的元数据，以及带有排程选项的**计算**部分。同一表单在新建项目时也充当项目向导。

## 打开方式

- **设置**（功能区标签）→ **项目**功能区组 → **项目信息**。
- 设置窗口（齿轮 ⚙）→ **常规**标签 → **项目信息...**
- **文件** → **项目信息** — Backstage 中的简化版本，只有元数据字段（没有计算部分）。

**应用**一次性提交所有更改；**取消**、**Esc**或点击窗口外部会放弃更改。**Enter** 效果与应用相同。

## 元数据

- **项目名称** — 标题栏和文档标签页中显示的名称。
- **描述** — 自由文本。
- **工程师**和**公司** — 自由文本；存储在 IFC 文件中。
- **开始日期** — 计算所依据的项目开始日期。
- **结束日期** — 信息性的项目结束日期。

## 计算

此项目的排程选项——它们随文件存储，而非随应用存储，因此会随文件一起传递到其他计算机。若在此处更改任何内容，计划会在**应用**后自动重新计算。

- **关键定义** — **总浮动 ≤ 阈值**（带**阈值（工作日）**，默认 0）或**最长路径**。
- **浮动计算** — **最小值（开始/结束）**（默认）、**开始浮动**或**结束浮动**。
- **开放式任务视为关键** — 将没有后续任务的任务标记为关键。
- **标记为接近关键** — 勾选后会显示额外的**阈值**（默认 2 个工作日；单位遵循工期显示设置，因此可能是小时）：浮动较小的任务会获得“接近关键”标记。
- **多条浮动路径** — 勾选后会显示**方法**（**自由浮动（剥离法）**或**总浮动（排序法）**）和**最大路径数**（默认 10）：计算随后会为最重要的浮动路径编号。
- **延迟日历** — 哪个日历用于计算关系的延时：**前置任务**（默认）、**后续任务**、**24 小时制**或**项目日历**。

如何解读这些结果，请参见 [Kritiek pad & geavanceerde analyse](docs://gids-kritiek-pad-analyse)。

## 项目向导（新建）

**新建**会以向导的形式打开同一个窗口（标题**新建项目**，按钮**创建**）。除元数据字段外，向导还包含：

- **阶段模板** — **空白**、**住宅建筑**或**公共建筑 / 改造**：为新项目填入一份阶段结构。
- **班次** — 仅在启用小时计划时可见：**白班**（默认）、**两班倒**、**三班倒**或 **24/7**。
- **假日集** — 生成项目日历：选择一个国家（适用时带地区和建筑业假期）、**无假日**，或**自定义…**——后者会在创建后立即打开日历对话框，供您手动组建日历。参见 [Kalenderdialoog](docs://ref-kalenderdialoog)。

向导中没有计算部分；创建后请通过上述任一入口进行设置。
