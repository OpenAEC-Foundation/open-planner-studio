# Externe koppelingen

**外部（跨项目）链接**窗口记录本项目中的一个任务与另一个项目文件中某个任务之间的依赖关系——例如一个必须在您开工前完成的场地工程项目。

## 打开方式

**关系**标签 → **外部链接…**按钮。必须恰好选中一个任务；否则会显示“请选择单个任务以添加外部链接。”

## 冻结的锚点

外部链接不会与源项目进行实时计算。添加时，源任务的相关日期（开始或完成，取决于方向和关系类型）会作为固定的**锚点日期**存储；计算会将该日期作为边界使用。如果之后源项目发生变化，在您**刷新**该链接之前，不会有任何内容随之移动。

## 两种途径

- **源文件** — 在**选择最近使用的文件**下选择一个文件；该文件以只读方式读取（“源文件以只读方式读取 — 不会作为文档打开。”）。然后从列表中选择**源任务**；锚点日期会自动从该任务读取并显示在底部。此途径需要桌面应用，且至少有一个最近文件。
- **手动（备用方式）** — 手头没有文件（或使用浏览器版本）：粘贴外部任务的**项目 ID**和**任务 ID**，可选填**任务名称**，并自行输入**锚点日期**。手动链接会标记为“已过期”，直到一次刷新真正找到源为止。

## 共用字段

- **方向** — **前置任务（外部 → 我）**：外部任务驱动我的任务；或**后续任务（我 → 外部）**：我的任务驱动外部任务。
- **关系类型** — FS、SS、FF 或 SF。
- **延迟（工作日）** — 在锚点基础上的等待时间（或负值：重叠）。

**添加链接**保存该链接（必填字段未填时不可用）；**取消**关闭窗口而不添加。

## 管理、刷新与源缺失

已有链接列在关系面板的**外部链接**下：

- 每个链接显示：源任务、类型、锚点，以及一旦源无法（再次）加载就会出现的**已过期**徽章——附带说明“源未加载 — 重新导入以刷新”。
- **刷新此链接** — 重新读取该单个链接的源文件并更新锚点。
- **刷新外部锚点** — 重新读取每个被引用的源文件，并更新所有锚点及过期状态。之后一行状态信息会报告刷新了多少个锚点，多少个仍处于过期状态。
- **移除** — 删除该链接。
- 刷新需要读取文件，因此仅在桌面应用中可用；浏览器版本会报告“读取源文件仅在桌面应用中可用；请使用手动备用方式。”
