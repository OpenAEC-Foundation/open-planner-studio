# Kritiek pad & geavanceerde analyse

每个计划都有一条最长的任务链，共同决定项目何时完成：关键路径。其之外的一切都有浮动——可以滑移而不影响结束日期的余地。本指南超越“哪些任务条是红色的”这一层面：总浮动/自由浮动/干扰浮动、接近关键的工作、多条同等关键的路径、hammock、硬钉住及其上游影响，以及项目之间的外部链接。

## 您将在此学到什么

- 解读关键路径，以及总浮动、自由浮动和干扰浮动之间的区别。
- 接近关键的工作：设置阈值并识别琥珀色标记。
- 同时存在多条关键路径：何时发生以及如何查看。
- 硬钉住及其对浮动的影响，包括在上游出现的负浮动。
- Hammock（工作量层级）：它们做什么、不做什么。
- 项目之间的外部链接：冻结的锚点、刷新，以及“源缺失”状态。
- 通过上下文菜单或功能区追踪路径。
- 项目设置中的**计算**部分。

跟随 [Nieuwbouw Appartementencomplex De Vaart](examples://showcase-appartementencomplex.ifc) 学习——这个拥有三座并行塔楼的大型“百科全书式”展示项目，几乎涵盖了本指南中的每个主题：多条关键路径、接近关键的工作、一个 hammock、一个硬钉住，以及指向独立源文件的外部链接。

## 解读关键路径

按 **F5**（或**计算**按钮）运行计划。屏幕底部的状态栏随即会显示，例如“关键路径：N 个任务，M 个工作日”——关键路径上的任务数量和总工期。在甘特图中，关键任务拥有自己的（红色）任务条颜色：没有浮动的任务，其每一天的延误都会直接推迟项目结束日期。

双击一个任务，在 **CPM 结果**部分查看确切数字：**最早开始**、**最早完成**、**最晚开始**、**最晚完成**、**总浮动**、**自由浮动**以及（在适用时）**干扰浮动**，加上该任务是否位于**关键路径**上。想把这些字段作为列显示在任务表格中？**视图 → 列…**并勾选它们。

### 总浮动、自由浮动和干扰浮动

- **总浮动** — 一个任务总共能滑移多少而不影响项目结束日期。零表示关键。
- **自由浮动** — 一个任务能滑移多少而不影响其紧接的下一个后续任务。可能小于总浮动：一个任务可能有一些总浮动，但只要它滑移一天，其直接的后续任务就已经跟着移动了（那个后续任务自身则有足够的浮动，不会影响结束日期）。
- **干扰浮动** — 两者之差（总浮动 − 自由浮动）：您浮动中不影响结束日期、但确实会“妨碍”某个后续任务的那部分。零表示自由浮动和总浮动相等——在您的浮动范围内滑移不会影响任何人。

## 接近关键的工作

总浮动很小但非零的任务是脆弱的：一个小挫折就会使其变成真正的关键任务。通过**项目信息 → 计算 → 标记为接近关键**打开此功能，设置一个以工作日（或小时，取决于您的工期显示设置）表示的**阈值**。每个总浮动大于零且小于或等于该阈值的任务，在甘特图中都会获得琥珀色的任务条颜色——介于关键的红色和浮动充裕的绿色之间。

大型展示项目将阈值设为 3 个工作日。**Tower C** 的最终检查因此恰好有 3 个工作日的总浮动——正好在阈值以内——而 **Tower A** 和 **Tower B** 中相同的最终检查则处于零浮动，属于真正的关键。除了一个略短的装修任务外，Tower C 在任务和工期上与另外两座塔楼完全相同；正是这一微小差异，恰好足以使其从关键变为接近关键。

## 多条关键路径

通常只存在一条最长的任务链，但也可能出现两条或更多条链长度完全相同——这时它们（全部）同等关键。开启**多条浮动路径**（**项目信息 → 计算**）以计算这一点：选择**方法**（**自由浮动（剥离法）**或**总浮动（排序法）**）和一个**最大路径数**。此后每个任务都会获得一个**浮动路径**编号（1 = 最关键）；没有浮动路径的任务不在任何已计算的路径上。

在大型展示项目中，Tower A 和 Tower B 在任务和工期上完全对称——它们恰好同时完成。一旦您开启**多条浮动路径**，结果中会看到不止一条路径（计算中 `criticalPaths.length` 大于 1）：不是单一的最长链，而是多条贯穿项目的同等关键的链条。这与“一条关键路径旁边有一些接近关键的工作”是不同的信号——它意味着这些路径中*任意*一条的延误都会同等程度地影响结束日期，因此您无法将注意力集中在单一链条上。

## 硬钉住及其对浮动的影响

**硬钉住**（MSO 或 MFO 限制上的**强制（钉住逻辑）**复选框）将任务固定在某个日期上，即使其前置任务在逻辑上与此矛盾。大型展示项目将其用于“Wegafzetting gemeente (vergunde stremmingsperiode)”（市政道路封闭，已获许可的封闭期）：市政府只允许在恰好那个已获许可的日期进行封闭，没有商量余地——网络逻辑必须围绕它调整。

难以看透的部分是其上游影响：如果被钉住任务的前置任务所需时间超过了钉住日期之前可用的时间，那些前置任务上就会出现**负浮动**。因此负浮动并不是计算错误：它是引擎告诉您“这条前置链条已经无法再容纳在钉住点允许的时间内”的方式。如果您在硬钉住的上游看到负浮动，问题不是“这里出了什么故障”，而是“这两者之中哪一个必须让步：钉住日期，还是其之前链条的工期”。

注意：在大型展示项目中，围绕“Wegafzetting gemeente”的整条链——包括被钉住的任务本身——早已全部完成（实际开始和完成，远早于状态日期）。正因如此，您会在整个第一阶段链条上看到一点残余的负浮动，包括在钉住任务本身上：这是已完成任务与状态日期结合的特征，而不是上文描述的“前置任务不再适配”的情形。要以纯粹形式看到那种情形：临时清除状态日期（**基线与进度**功能区组，**清除状态日期**按钮）并重新计算——钉住任务本身随即回到零总浮动，只有当您刻意使前置链条比钉住日期之前的可用空间更长时，负浮动才会出现。

## Hammock（工作量层级）

**Hammock**（属性面板中的**Hammock（派生工期）**复选框）是一个自身没有工期输入的任务：其开始和完成自动由其自身的关系推导而来。传入的 **FS**/**SS** 关系提供**开始驱动**（最早的开始），传入的 **FF**/**SF** 关系提供**结束驱动**（最晚的完成）——一旦您勾选 hammock 复选框，面板会以只读方式显示两者，以便您准确看清是哪些任务决定了跨度。若没有结束驱动，跨度会回退为零长度，面板中会显示警告。

Hammock 做的事情是：以一种概括性任务条的形式，展示一段工作的完整跨度，而无需您自己维护工期——适用于例如“监理”或“工地总管理费”一类字面上跨越整个底层工作周期的内容。Hammock 不做的事情是：它不携带任何影响 CPM 计算的资源或逻辑——它是一个派生视图，而不是驱动性任务。大型展示项目将其用于“Ruwbouw toren A (LOE)”（A 塔楼主体结构，LOE）：一个 hammock，一旦 A 塔楼的第一个真正主体结构任务开始就开始，一旦最后一个完成就完成，其本身不占据中间的任何位置。

## 项目之间的外部链接

大型项目有时由多个各自独立管理的子计划组成——例如您自己的主计划和另一位承包商管理的场地工程包。**外部链接**（通过**关系**标签上的按钮打开的**外部（跨项目）链接**窗口）记录与另一个此类文件中某个任务的关系，而无需将该文件作为文档打开。

您可以从最近文件中选择一个**源文件**（以只读方式读取，永不作为文档打开），或者如果手头没有源文件，可以填写**手动**方式，输入项目 ID、任务 ID 和锚点日期。**锚点日期**——您链接时源任务所处的日期——会冻结在您自己的文件中；如果源项目发生变化，该日期不会自动跟随更新。

想知道源文件自那以后是否已更新？前往**关系**标签的**外部链接**部分，点击**刷新此链接**（针对单个链接）或**刷新外部锚点**（一次性刷新全部）以重新读取源文件并更新锚点。如果源文件不可用——已移动、已重命名，或从未随附——链接会显示**已过期**标签，并附带提示“源未加载 — 重新导入以刷新”：这时应用无法自行验证冻结的锚点是否仍然成立。

大型展示项目刻意演示了正是这最后一种情况：任务“Bestrating parkeerterrein”（停车场铺装）链接到一个来自场地工程分包商的源文件，该文件刻意*未*随示例一起提供。打开该任务，您会看到该链接列出状态为“已过期”——这真实地演示了当外部源文件不再可用时会发生什么，而不是一个总是完美刷新的链接。

## 追踪路径

想确切看到哪些任务在上游和下游影响给定任务？右键点击该任务并选择**追踪路径**（或**停止路径追踪**以再次关闭它）——这会一次性高亮整条前置任务和后续任务链。若需要更精确的操作，功能区（**计划**或**关系**标签，**路径追踪**功能区组）提供一对独立按钮**前置任务**/**后续任务**：都关闭时不显示任何内容，打开一个则显示该方向，两个都打开则等同于上下文菜单命令。追踪还会区分所有逻辑上相连的任务与实际**驱动**该日期的任务（与关系表格中显示的相同“驱动”关系）——因此您看到的不仅是什么相连，还有什么在实际起主导作用。

## 计算设置

**项目信息**中的**计算**部分（Backstage → 项目信息，或**项目信息**窗口）收集属于此特定项目的计算选项——它们属于文件，而非应用，因此打开同一文件的同事会得到相同的结果：

- **关键定义** — **总浮动 ≤ 阈值**（默认阈值 0）或**最长路径**，后者根据网络中最长的链条来标记关键任务，与其浮动数值无关。
- **浮动计算** — 对同时具有开始侧和完成侧的任务，如何确定总浮动：**最小值（开始/结束）**（默认）、**开始浮动**或**结束浮动**。
- **开放式任务视为关键** — 自动将没有后续任务的任务视为关键。
- **标记为接近关键**，配合**阈值**（见上文）。
- **多条浮动路径**，配合**方法**和**最大路径数**（见上文）。
- **延迟日历** — 以工作日表示的延时使用哪个日历：**前置任务**的、**后续任务**的、始终使用**24 小时制**，或**项目日历**。
