# Herstellen na een crash

桌面应用会自动保留您工作的恢复快照。如果应用意外关闭（崩溃、断电），它会在下次启动时提出恢复这些工作的选项。

## 自动保存的工作方式

- 每次更改之后不久（不到一秒），应用会将每个打开文档的快照写入其自己的数据文件夹——涵盖所有打开的标签页，包括从未保存过的文档。
- 这不能替代保存：您的项目文件本身不会因此改变。因此仍需自行使用 Ctrl+S 保存工作。
- 快照会在您在恢复窗口中做出选择后（**恢复**或**不恢复**）被清理。
- **仅限桌面应用。** 浏览器版本没有自动保存也没有恢复功能——请在浏览器中自行定期保存。

## “恢复未保存的工作”窗口

在启动时发现快照会出现：“Open Planner Studio 未正常关闭。以下文档存在可恢复的未保存更改：”对每个文档会显示：

- **名称**（文件名或项目名；未命名时为“未命名项目”）；
- **文件路径**（如果该文档曾经保存过）；
- 快照中的**任务数量**；
- **保存时间** — 最新快照的时间。

## 可选择的操作

- **恢复**（或 **Enter**） — 所有列出的文档会作为打开的标签页重新出现。此后它们算作未保存状态：请自行保存它们。
- **不恢复** — 快照会被丢弃；您从一个空项目开始。
- **关闭图标**、**Esc**或点击窗口外部 — 安全地推迟决定：不会丢弃任何内容，也不会恢复任何内容；下次启动时会再次出现该问题。
