# 보고서 및 인쇄

일정은 공유할 수 있어야 완성됩니다 — 현장 회의를 위한 종이로, 프레젠테이션의 이미지로, 또는
다가오는 것과 이미 이동한 것의 개요로. 그것이 **보고서** 탭이 존재하는 이유이며, 세 가지 보고서
유형과 인쇄 미리보기가 있습니다.

## 여기서 배울 내용

- **보고서** 탭의 세 가지 보고서 유형: 간트 인쇄, 마일스톤 개요, 차이.
- 인쇄 미리보기 작동 방식: 용지 크기, 방향, 켜고 끌 수 있는 요소.
- 보고서를 실제로 인쇄하거나 파일로 저장하는 방법.
- 이 앱에서 **Ctrl+P**가 하는 일.

## 보고서 화면에 도달하기

같은 화면으로 가는 세 가지 방법이 있습니다: **보고서** 리본 탭 클릭, **백스테이지 → 인쇄**(보고서
화면을 직접 엽니다)로 이동, 또는 **Ctrl+P**를 누릅니다. 세 방법 모두 같은 곳으로 이어집니다 —
별도의 "인쇄" 대화상자는 없습니다; 보고서 화면 *자체가* 인쇄 미리보기입니다.

화면은 두 열로 나뉩니다: 상단에 **보고서 유형** 선택기가 있는 왼쪽 설정 패널과, 왼쪽에서 설정을
변경하는 즉시 업데이트되는 오른쪽 실시간 미리보기.

## 세 가지 보고서 유형

### 간트 인쇄

간트 막대의 완전하고 형식화된 인쇄물입니다 — 설정 블록이 있는 유일한 보고서 유형입니다:

- **용지**: A4, A3 또는 A1.
- **방향**: 가로 또는 세로.
- **용지에 자동 맞춤**(켜짐 = 일정이 선택한 크기에 자동으로 맞춰짐) 또는 자동 맞춤을 끄면 수동
  **확대/축소** 슬라이더.
- **바에 작업 이름 표시**, **진행률 표시**, **크리티컬 패스**, **여유 표시**, **종속성**, **주말**,
  **범례**의 토글.
- **회사** 필드(프로젝트 설정에서 자동으로 채워지지만 여기서 별도로 편집 가능)와 **작성자**(프로젝트
  정보에서 가져온 읽기 전용).

위의 요약 블록은 프로젝트의 작업, 세부 작업, 크리티컬 작업, 관계의 실시간 개수를 보여줍니다.

### 마일스톤 개요

프로젝트의 모든 마일스톤 표: WBS, 이름, 종류(자동/시작/완료), 날짜, 근본 제약 조건이나 기한,
여유, 마일스톤이 필수인지 여부, 상태(일정대로 / 임계 / 지연). 요약 블록은 총 마일스톤 수, 몇 개가
필수이고 몇 개가 지연되었는지 보여줍니다. 이 보고서에는 용지 크기/방향 설정이 없습니다 — 표시된
그대로 표를 인쇄합니다.

### 차이

현재 일정을 활성 기준선과 비교합니다: 기준선 시작/종료 대비 현재 시작/종료, 시작과 종료의
근무일 차이, 작업별 상태(일정대로 / 지연 / 조기 / 신규 / 삭제됨). 활성 기준선이 없으면 화면은
빈 보고서를 표시하는 대신 이를 명시적으로 알립니다. 요약 블록은 프로젝트 종료일의 근무일 이동도
있으면 표시합니다. 이 보고서가 유용한 정보를 알려주려면 먼저 기준선을 기록하는 방법에 대한 가이드
[기준선 및 진행](docs://gids-baselines-voortgang)을 참조하세요.

## 인쇄 및 내보내기

설정 패널 하단에는 항상 **인쇄...** 버튼이 있습니다 — 이는 보고서가 포함된 별도의 인쇄 창을 열고
즉시 브라우저/OS 인쇄 대화상자를 트리거합니다. 간트 보고서의 경우 그 창은 선택한 용지 크기와
방향을 사용합니다; 마일스톤과 차이 보고서는 표시된 그대로 표를 인쇄합니다.

간트 보고서만 **PDF 내보내기** 버튼도 있습니다. 이는 현재 미리보기를 실제 PDF 파일로 저장합니다
(파일명은 `-planning.pdf`로 끝남) — 선택한 용지 크기와 방향의 물리적 크기에 맞춘 한 페이지입니다.
PDF 파일은 **벡터 기반**입니다: 막대, 선, 텍스트는 단일 내장 이미지가 아니라 PDF 그리기
명령으로 저장되므로, 어떤 확대/축소 수준에서도 선명하며 어떤 PDF 뷰어에서도 텍스트를 선택하고
검색할 수 있습니다. 이는 라틴, 키릴, 그리스 문자에 적용됩니다; 프로젝트에 중국어, 일본어, 한국어,
아랍어 또는 페르시아어 텍스트가 포함되어 있으면, 내보내기는 해당 텍스트에 대해 자동으로 래스터
이미지로 대체됩니다 — 여전히 올바르게 표시되지만 선택하거나 검색할 수는 없습니다. 시스템 인쇄
대화상자를 거치지 않고 이메일이나 보관용으로 편리합니다. 직접 인쇄하거나(또는 위에서 구성한 것과
다른 용지 크기를 선택하기 위해 시스템 대화상자를 통해 PDF로 저장하려면) **인쇄...**를 사용하세요.

## 실제 활용에서의 보고서

각 보고서 유형은 서로 다른 대화를 위해 사용됩니다:

- **간트 보고서**는 고전적인 현장 회의 배포용 자료입니다: 크리티컬 패스가 강조되고, 크리티컬하지
  않은 막대에는 여유가 보이며, 범례가 각 색상의 의미를 설명합니다. 청중이 아직 일정을 모른다면
  **바에 작업 이름 표시**와 **진행률 표시**를 켜세요; 별도의 작업 목록이 함께 배포된다면 A1에서
  깔끔한 개요를 위해 이를 끄세요.
- **마일스톤 개요**는 수십 개의 작업 행을 넘기지 않고 중요한 날짜만 원하는 사람을 위한 것입니다 —
  예를 들어 필수 인도 날짜가 지켜지는지 주로 알고 싶은 고객. 표에서 마일스톤 이름 앞의 ◆ 기호는
  **필수** 마일스톤을 표시합니다.
- **차이 보고서**는 방향을 수정하는 대화입니다: 기준선 대비 어떤 작업이 밀리고 있으며, 며칠의
  근무일만큼인지. 이 보고서를 실제로 확인하려면 쇼케이스
  [Nieuwbouw Appartementencomplex De Vaart](examples://showcase-appartementencomplex.ifc)를
  참조하세요, 이는 자체 진행률과 상태 날짜를 가진 두 개의 기준선(계약 기준선과 변경 지시 후
  재기준선)이 있습니다 — 기준선과 현재 일정 사이에 실제 차이가 생기면 Δ 열이 어떻게 채워지는지
  보여주는 좋은 예입니다.

오른쪽의 실시간 미리보기는 왼쪽의 설정을 변경할 때마다 새로 고쳐집니다 — 별도의 "새로 고침"
버튼은 없으며, 인쇄 시점에만 계산되는 것은 아무것도 없습니다.

## 더 읽어보기

- 기준선이 기록되기 전까지 차이 보고서는 비교할 것이 없습니다 — 가이드
  [기준선 및 진행](docs://gids-baselines-voortgang)을 읽어보세요.
- 간트 보고서에 표시되는 크리티컬 패스와 여유는 간트 뷰 자체와 같은 계산에서 나옵니다 — 이를
  읽는 방법은 가이드 [크리티컬 패스 및 고급 분석](docs://gids-kritiek-pad-analyse)을 읽어보세요.
