# 계획 및 WBS

일정은 작업 구조에서 시작합니다: 어떤 작업이 존재하고, 어떻게 단계로 나뉘며, 어떤 순간이 마일스톤을 가질 만큼 중요할까요? 이 가이드는 [빠른 시작](docs://quick-start) 가이드보다 이 기초를 더 깊이 다룹니다 — 여기서는 들여쓰기하는 *방법*뿐만 아니라 요약 작업이 실제로 무엇을 하는지, 세 가지 마일스톤 종류가 어떻게 다른지, 작업에 고유한 코드와 필드를 부여하는 방법, 작업별로 메모를 유지하는 방법도 배웁니다.

## 여기서 배울 내용

- 들여쓰기와 요약 작업을 사용하여 작업 구조(WBS) 만들기.
- 다시 들여쓰기하지 않고 같은 레벨 내에서 작업 이동하기.
- 세 가지 마일스톤 종류와 계약상 순간을 위한 별도의 필수 플래그.
- **코드 및 필드** 창을 통해 작업 코드와 사용자 정의 필드를 관리하고 이를 기준으로 그룹화하기.
- 메모(작업별 체크리스트)를 사용하여 미해결 항목 추적하기.

완전한 예제를 따라가고 싶으신가요? **파일 → 예제**를 통해 [Verbouwing & Aanbouw Eengezinswoning](examples://showcase-verbouwing-eengezinswoning.ifc)를 여세요 — 하위 작업이 있는 "1. Voorbereiding"(준비) / "2. Fundering & ruwbouw"(기초 및 골조 공사) / "3. Afbouw"(마감) / "4. Oplevering"(인도) 단계 구성이 정확히 아래에서 설명하는 구조입니다.

## 작업 구조 만들기

작업의 평면 목록은 그것들이 어떻게 관련되어 있는지 아무것도 말해주지 않습니다. 한 작업을 다른 작업 아래에 들여쓰기하면 트리 구조(WBS — Work Breakdown Structure)를 만들게 됩니다: 상위 작업이 자동으로 **요약 작업**이 됩니다.

1. 구조에서 더 깊은 곳에 배치하려는 작업을 선택합니다.
2. **Alt+→**를 눌러 들여쓰기합니다. 같은 동작을 위한 두 번째 단축키가 있습니다: **Alt+Shift+→** — 키보드 레이아웃이 이미 Alt+→를 다른 용도로 사용하는 경우 편리합니다. 둘 다 정확히 같은 동작을 합니다.
3. 마우스로 작업하는 것을 선호하시나요? 작업을 마우스 오른쪽 버튼으로 클릭하고 컨텍스트 메뉴에서 **들여쓰기**를 선택하세요.
4. 한 레벨 너무 멀리 갔나요? **Alt+←**(또는 마우스 오른쪽 버튼 클릭 → **내어쓰기**)는 작업을 한 레벨 뒤로 이동시킵니다.
5. 완전히 새로운 하위 작업에는 더 빠른 경로가 있습니다: 상위 작업을 마우스 오른쪽 버튼으로 클릭하고 **하위 작업 추가**를 선택하세요. 이는 먼저 작업을 추가한 후 별도로 들여쓰기하는 대신, 이미 들여쓰기된 새 작업을 한 단계로 만듭니다.

작업이 최소 하나의 하위 작업을 갖는 즉시 자동으로 요약 작업이 됩니다: 간트 차트의 막대는 그 아래 모든 하위 작업의 가장 이른 시작부터 가장 늦은 종료까지 전체 기간에 걸쳐 있게 되며, 자체 기간과 날짜는 더 이상 독립적으로 설정할 수 없습니다. 따라서 요약 작업은 항상 파생 값이며 직접 입력하는 일정이 아닙니다 — 하위 작업을 삭제하거나 이동하면 요약 작업의 막대가 자동으로 조정됩니다.

### 다시 들여쓰기하지 않고 작업 이동하기

작업의 레벨을 변경(들여쓰기/내어쓰기)하는 것 외에도, 구조 자체를 바꾸지 않고 같은 레벨 내에서 작업의 위치를 바꿀 수도 있습니다:

- **Alt+↑**는 선택한 작업을 위로 이동하여 현재 위에 있는 작업 위로 옮깁니다.
- **Alt+↓**는 작업을 아래로 이동시킵니다.

이는 트리의 어떤 레벨에서도 작동합니다: 단계 작업을 이동하면 그 모든 하위 작업이 자동으로 함께 이동합니다.

## 마일스톤 종류

마일스톤은 기간이 없이 순간을 표시하는 작업입니다 — 시작, 인도, 검사 등. Open Planner Studio는 마일스톤을 추가하는 세 가지 방법을 제공하며, 모두 **작업** 리본 그룹에서 **마일스톤** 버튼 옆의 화살표를 통해 이루어집니다:

- **시작 마일스톤** — 단계나 프로젝트의 시작을 표시합니다.
- **완료 마일스톤** — 완료를 표시합니다(예: 인도).
- **검사 지점(필수)** — 실제로는 **필수(계약상)** 플래그가 이미 체크되고 유형이 **검사**로 직접 설정된 완료 마일스톤이므로, 검사 순간은 처음부터 계약상 의무이자 검사임을 알아볼 수 있습니다.

단축키 **Ctrl+M**을 선호하시나요? 이는 일반 마일스톤("새 마일스톤")을 제공하며 이후 직접 이름과 유형을 지정합니다.

**마일스톤** 체크박스가 켜진 마일스톤을 선택하면 속성 패널에서도 동일한 구분을 볼 수 있습니다: **마일스톤 종류** 필드는 **자동**, **시작 마일스톤** 또는 **완료 마일스톤**을 제공합니다. "자동"은 마일스톤에 뚜렷한 시작이나 종료 특성이 없을 때 관계에 따라 스케줄링 엔진이 마일스톤의 동작 방식을 결정하도록 합니다. 별도로 **필수(계약상)** 체크박스가 있습니다: 이는 시작이든 완료 마일스톤이든 관계없이 마일스톤을 계약상 구속력이 있는 것으로 표시합니다. 이렇게 하면 예를 들어 시작 마일스톤도 필수로 만들 수 있고, **검사 지점**처럼 필수 완료 마일스톤을 클릭 한 번으로 설정할 수 있습니다.

## 코드 및 필드: 작업 코드와 사용자 정의 필드

더 큰 일정에는 WBS에 맞지 않는 추가 차원이 곧 필요해집니다: 어떤 유닛, 어떤 공종, 어떤 시공사. 이를 위한 것이 **작업 코드**와 **사용자 정의 필드**이며, 둘 다 **코드 및 필드** 창을 통해 관리됩니다(**계획** 탭의 **구조** 리본 그룹, **코드 및 필드**라는 버튼).

- **작업 코드**는 값 목록을 가진 자유롭게 정의 가능한 차원(예: "위치" 또는 "공종")입니다 — 각 값에는 **코드**, **설명**, **색상**이 있습니다. 작업은 코드 유형당 최대 하나의 값을 가질 수 있습니다. 새 차원을 시작하려면 **코드 유형 추가**를, 가능한 값을 구성하려면 **값 추가**를 사용하세요.
- **사용자 정의 필드**는 작업 표에 열로 표시되고 작업별로 채울 수 있는, 여러분만의 형식이 지정된 필드입니다 — **텍스트**, **숫자**, **정수**, **비용**, **날짜** 또는 **예/아니오**. "시공사"(텍스트)나 "허가 취득"(예/아니오) 같은 필드를 생각해보세요.

생성한 후에는 작업 표의 열(필요하면 **보기 → 열…**을 통해 먼저 표시)이나 작업의 속성 패널을 통해 작업 코드를 할당하거나 사용자 정의 필드를 채웁니다.

### 코드와 필드로 그룹화하기

작업 코드와 사용자 정의 필드는 그것을 기준으로 그룹화할 때 정말로 진가를 발휘합니다: 리본 탭 **보기**로 이동하여 **그룹화**를 열고 **필드** 아래에서 묶을 작업 코드나 사용자 정의 필드를 선택하세요. 그러면 작업 표는 WBS 트리 대신 그룹 헤더를 표시합니다 — 예를 들어 단계 구성 전체에서 유닛별 또는 공종별 모든 작업을 함께 보는 데 편리합니다. 최대 두 개의 그룹화 레벨을 동시에 설정할 수 있습니다(예: 먼저 유닛별, 그다음 공종별).

## 메모: 작업별 체크리스트

모든 작업에는 속성 패널에 **메모** 섹션이 있습니다 — 본질적으로 작업에 붙어 있는 작은 체크리스트입니다. 이는 일정 날짜에 맞지 않는 종류의 느슨한 조치 항목을 위한 것입니다: "시공사와 아직 확인해야 함", "아직 자재를 주문해야 함", "도면 v2 대기 중".

1. **+ 메모 추가**를 클릭합니다. 텍스트 필드에 포커스가 있는 새 빈 행이 나타납니다.
2. 메모 텍스트를 입력합니다.
3. 항목이 처리되면 체크박스를 체크하세요 — 텍스트에 취소선이 그어지지만 메모는 계속 표시됩니다(삭제가 아니라 완료로 표시), 그래야 작업의 이력을 계속 읽을 수 있습니다.
4. 휴지통 아이콘을 사용하여 메모를 영구적으로 제거합니다.

메모는 순전히 정보 제공용입니다: 일정이나 계산에 영향을 주지 않으므로, 날짜나 기간으로 표현할 수 없는 비고를 위한 올바른 도구입니다. 중간 규모 예제 "Nieuwbouw 6 Rijwoningen De Akkers"(**파일 → 예제**의 *aantekeningen*/메모 태그)에서 미해결 및 완료된 메모가 섞여 있는 실제 사례를 확인해보세요.
