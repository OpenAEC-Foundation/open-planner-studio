# 확장 프로그램 관리 및 설치

확장 프로그램은 추가 가져오기 형식이나 사용자 지정 리본 버튼 같은 기능을 앱에 더합니다. 이는 앱 수준입니다: 프로젝트 파일이 아니라 이 기기의 이 설치본에 속합니다.

## 열기

**파일** → **확장 프로그램**(백스테이지). 상단에는 **ZIP**과 **JS** 버튼 옆에 **설치됨**과 **찾아보기** 두 개의 탭이 있고, 아래에 검색 필드(**확장 프로그램 검색...**)가 있습니다.

## 설치됨

확장 프로그램당 카드 하나에 이름, 버전, 카테고리, 설명, 작성자, 그리고:

- **켜기/끄기 토글** — 제거하지 않고 확장 프로그램을 사용/사용 안 함으로 전환합니다.
- **제거** — 완전히 제거하려면 **확인**을 한 번 더 클릭하세요.

로드에 실패한 확장 프로그램은 카드에 오류 메시지를 표시합니다. 확장 프로그램이 없으면 탭에 "아직 설치된 확장 프로그램이 없습니다."라고 표시됩니다.

## 찾아보기 (카탈로그)

**찾아보기** 탭은 온라인 확장 프로그램 카탈로그를 가져옵니다(인터넷 연결 필요). 각 카탈로그 항목은 **설치** 버튼이 있는 카드입니다; 이미 설치된 확장 프로그램은 **설치됨** 배지를 표시합니다. 로드에 실패하면 **다시 시도** 버튼과 함께 오류 메시지가 나타납니다.

## 파일에서 설치

- **ZIP** — 확장 프로그램 ZIP(`manifest.json` + `main.js` 포함)을 설치합니다.
- **JS** — 매니페스트가 내장된 단일 `.js` 파일을 설치합니다.

설치 후 확장 프로그램은 즉시 사용 설정되며 리본 버튼이 있다면 바로 나타납니다.

## 확장 프로그램을 통한 가져오기

**파일** → **가져오기**는 설치된 확장 프로그램이 제공하는 가져오기 형식을 나열합니다; 형식을 클릭하고 파일을 선택하세요. 가져오기 확장 프로그램이 없으면 페이지에 "가져오기 확장 프로그램이 없습니다. 확장 프로그램에서 추가하세요."라고 표시됩니다. 내장 가져오기 형식(CSV, MS Project, P6)은 이와 별개입니다 — [가져오기/내보내기](docs://gids-import-export)를 참조하세요.

## 직접 확장 프로그램 작성하기

확장 프로그램 작성자를 위한 가이드(매니페스트, API, 권한)는 저장소에 있습니다: `github.com/OpenAEC-Foundation/open-planner-studio`, 파일 `docs/extensions.md`.
