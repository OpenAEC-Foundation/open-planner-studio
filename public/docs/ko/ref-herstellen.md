# 충돌 후 복구

데스크톱 앱은 작업 내용의 복구 스냅샷을 자동으로 보관합니다. 앱이 예기치 않게 종료되면(충돌, 정전) 다음 실행 시 해당 작업을 되살릴지 물어봅니다.

## 자동 저장 작동 방식

- 변경할 때마다 잠시 후(1초 이내) 앱은 열려 있는 각 문서의 스냅샷을 자체 데이터 폴더에 씁니다 — 열려 있는 모든 탭에 대해, 한 번도 저장되지 않은 문서를 포함해서.
- 이것은 저장을 대체하지 않습니다: 프로젝트 파일 자체는 변경되지 않습니다. 그러니 Ctrl+S로 작업 내용을 계속 직접 저장하세요.
- 스냅샷은 복구 창에서 선택(**복원** 또는 **복원 안 함**)을 하는 즉시 정리됩니다.
- **데스크톱 앱 전용.** 브라우저 버전에는 자동 저장과 복구가 없습니다 — 그곳에서는 직접 정기적으로 저장하세요.

## "저장되지 않은 작업 복원" 창

스냅샷이 발견되면 시작 시 나타납니다: "Open Planner Studio가 정상적으로 종료되지 않았습니다. 다음 문서에는 복원할 수 있는 저장되지 않은 변경 사항이 있습니다:" 각 문서에 대해 다음이 표시됩니다:

- **이름**(파일 이름 또는 프로젝트 이름; 이름이 없으면 "제목 없는 프로젝트");
- 문서가 저장된 적이 있다면 **파일 경로**;
- 스냅샷의 **작업 수**;
- **저장 시각** — 최신 스냅샷의 시각.

## 선택

- **복원**(또는 **Enter**) — 나열된 모든 문서가 열린 탭으로 돌아옵니다. 이후 이들은 저장되지 않은 것으로 간주됩니다: 직접 저장하세요.
- **복원 안 함** — 스냅샷이 삭제되고 빈 프로젝트로 시작됩니다.
- **닫기 버튼**, **Esc** 또는 창 밖 클릭 — 안전하게 미룹니다: 아무것도 삭제되거나 복원되지 않으며, 다음 실행 시 질문이 다시 나타납니다.
