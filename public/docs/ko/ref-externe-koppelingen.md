# 외부 링크

**외부(프로젝트 간) 링크** 창은 이 프로젝트의 작업과 다른 프로젝트 파일의 작업 사이의 종속성을 기록합니다 — 예를 들어 시작 전에 완료되어야 하는 부지 공사 프로젝트.

## 열기

**관계** 탭 → **외부 링크…** 버튼. 정확히 하나의 작업을 선택해야 합니다; 그렇지 않으면 "외부 링크를 추가하려면 작업 하나를 선택하세요."가 표시됩니다.

## 고정된 앵커

외부 링크는 소스 프로젝트에 대해 실시간으로 계산하지 않습니다. 추가할 때 소스 작업의 관련 날짜(방향과 관계 유형에 따라 시작 또는 종료)가 고정된 **앵커 날짜**로 저장됩니다; 계산은 이 날짜를 경계로 사용합니다. 이후 소스 프로젝트가 변경되어도 링크를 **새로 고침**하기 전까지는 아무것도 함께 이동하지 않습니다.

## 두 가지 경로

- **소스 파일** — **최근 파일 선택** 아래에서 파일을 선택하세요; 읽기 전용으로 읽어옵니다("소스 파일은 읽기 전용으로 읽어옵니다 — 문서로 열리지 않습니다."). 그다음 목록에서 **소스 작업**을 선택하세요; 앵커 날짜는 해당 작업에서 자동으로 읽혀 하단에 표시됩니다. 이 경로는 데스크톱 앱과 최소 하나의 최근 파일이 필요합니다.
- **수동(대체)** — 파일이 없는 경우(또는 브라우저 버전): 외부 작업의 **프로젝트 ID**와 **작업 ID**를 붙여넣고, 선택적으로 **작업 이름**을 입력하고, **앵커 날짜**를 직접 입력하세요. 수동 링크는 실제로 새로 고침이 소스를 찾을 때까지 "오래됨"으로 표시됩니다.

## 공통 필드

- **방향** — **선행(외부 → 나)**: 외부 작업이 내 작업을 주도합니다; 또는 **후행(나 → 외부)**: 내 작업이 외부 작업을 주도합니다.
- **관계 유형** — FS, SS, FF 또는 SF.
- **지연(작업일)** — 앵커에 더해지는 대기 시간(음수면 겹침).

**링크 추가**는 링크를 저장합니다(필수 필드가 채워질 때까지 비활성화); **취소**는 추가하지 않고 닫습니다.

## 관리, 새로 고침 및 소스 누락

기존 링크는 관계 패널의 **외부 링크** 아래에 나열됩니다:

- 링크별로: 소스 작업, 유형, 앵커, 그리고 소스를 (더 이상) 로드할 수 없게 되면 나타나는 **오래됨** 배지 — "소스가 로드되지 않음 — 새로 고치려면 다시 가져오세요"라는 설명과 함께.
- **이 링크 새로 고침** — 이 하나의 링크의 소스 파일을 다시 읽고 앵커를 업데이트합니다.
- **외부 앵커 새로 고침** — 참조된 모든 소스 파일을 다시 읽고 모든 앵커와 오래됨 상태를 업데이트합니다. 이후 상태 표시줄에 몇 개의 앵커가 새로 고쳐졌고 몇 개가 오래됨으로 남았는지 보고합니다.
- **제거** — 링크를 삭제합니다.
- 새로 고침은 파일을 읽으므로 데스크톱 앱에서만 작동합니다; 브라우저 버전은 "소스 파일 읽기는 데스크톱 앱에서만 가능합니다. 수동 대체 방법을 사용하세요."라고 보고합니다.
