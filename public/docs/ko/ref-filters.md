# 필터

**필터** 창은 어떤 작업이 표시되는지 — 간트와 테이블 탭 모두에서 — 제어합니다. 필터는 규칙(필드 + 연산자 + 값)으로 구성되며, 선택적으로 그룹으로 묶을 수 있습니다.

## 열기

**보기** → 리본 그룹 **표시** → **필터…**. 필터가 활성인 동안 버튼이 강조 표시된 상태로 유지됩니다. **Esc**, 닫기 버튼 또는 창 밖 클릭으로 적용하지 않고 닫힙니다.

## 그룹: 모두 또는 하나

각 그룹 상단에서 해당 규칙을 어떻게 결합할지 선택합니다:

- **다음 모두 (AND)** — 작업이 모든 규칙과 일치해야 합니다.
- **다음 중 하나 (OR)** — 하나의 규칙만 일치해도 충분합니다.

**+ 규칙**은 규칙을 추가합니다; **+ 그룹**(최상위 레벨만)은 중첩 그룹을 추가하여 AND와 OR을 조합할 수 있게 합니다 — 예를 들어 "크리티컬이 예 AND (유형이 건설 OR 유형이 설치)". 규칙이 없으면 창에 "아직 규칙이 없습니다 — 이 필터는 모든 항목과 일치합니다."라고 표시됩니다.

## 규칙: 필드, 연산자, 값

- **필드** — 모든 작업 필드: WBS, 작업 이름, 기간, 시작, 종료, 유형, 크리티컬, 총 여유, 진행률, 마일스톤, 자유 여유, 간섭 여유, 준임계, 여유 경로, 그리고 자원과 프로젝트의 작업 코드 및 사용자 정의 필드.
- **연산자** — 필드 유형에 따라 달라집니다:
- 텍스트: **같음**, **같지 않음**, **포함**, **다음으로 시작**, **비어 있음**;
- 숫자와 날짜: 추가로 **보다 작음**, **이하**, **보다 큼**, **이상**, **사이**(**시작**/**종료** 포함);
- 예/아니오 필드(크리티컬, 마일스톤 등): **예**/**아니오** 선택;
- 선택 필드(유형이나 작업 코드 등): **다음 중 하나**, 선택 가능한 값과 함께.
- **값** — 입력은 필드 유형을 따릅니다(텍스트 상자, 숫자, 날짜 또는 선택기); **비어 있음**에는 값 입력이 없습니다.

규칙 뒤의 휴지통 아이콘은 해당 규칙을 제거합니다; 중첩 그룹 오른쪽 상단의 X는 그룹 전체를 제거합니다.

## 적용, 취소, 지우기

- **적용**은 필터를 활성화하고 창을 닫습니다. 규칙이 없는 필터는 "필터 없음"으로 간주됩니다.
- **취소**는 변경 사항을 적용하지 않고 닫습니다.
- **지우기**는 활성 필터를 즉시 끄고 편집기를 비웁니다.

활성 필터는 저장된 레이아웃의 일부입니다 — [레이아웃 저장 및 불러오기](docs://ref-layouts)를 참조하세요.
