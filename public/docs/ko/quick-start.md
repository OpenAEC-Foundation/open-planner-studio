# 10분 만에 완성하는 첫 일정

이 가이드는 약 10분 안에 빈 프로젝트에서 완전히 계산된 건설 일정으로 안내합니다: 작업 추가, 작업 구조 만들기, 관계 추가, 계산 및 저장. 사전 이론 없이 — Open Planner Studio에서 실제로 볼 수 있는 정확한 버튼과 메뉴를 사용하여 단계별로 직접 해봅니다.

## 무엇을 할 것인가

1. 새 프로젝트 만들기.
2. 작업 추가하기 — 리본, 작업 표, 간트 차트를 통해.
3. 들여쓰기로 작업을 구조(WBS)에 넣기.
4. 작업 간 관계 추가하기.
5. 일정 계산하기.
6. 결과 읽기: 크리티컬 패스와 여유.
7. 저장하기.

먼저 목표 지점을 보고 싶으신가요? **파일 → 예제**를 통해 예제 프로젝트 [Verbouwing & Aanbouw Eengezinswoning](examples://showcase-verbouwing-eengezinswoning.ifc)를 여세요. (예제 이름은 프로젝트에 번들로 제공된 대로 네덜란드어로 표시됩니다.) 아래 단계 대부분을 이미 보여주는 작고 읽기 쉬운 일정입니다 — 비교를 위해 이 문서 옆에 열어두면 편리합니다.

아래 내용은 데스크톱 앱과 브라우저 버전에서 동일하게 작동합니다: 같은 버튼, 같은 메뉴, 같은 단축키.

## 1단계 — 새 프로젝트 만들기

1. 리본 탭 **파일**을 클릭합니다. 파일 화면이 열립니다.
2. **새로 만들기**를 클릭합니다(이미 다른 프로젝트에서 작업 중이라면 단축키 **Ctrl+N** 사용). **새 프로젝트** 대화상자가 나타납니다.
3. **프로젝트 이름**을 입력합니다(예: "나의 첫 일정") 그리고 **시작일**을 확인합니다 — 기본값은 오늘입니다.
4. **공정 단계 템플릿**으로 **비어 있음**을 선택합니다. **주택 건설**과 **비주거용 건축 / 리노베이션** 템플릿은 몇 가지 단계 작업을 미리 설정해 주지만, 이 연습에서는 모든 단계를 인식할 수 있도록 직접 만들어 볼 것입니다.
5. 달력 옵션을 기본값으로 두고 **만들기**를 클릭합니다.

이제 빈 프로젝트가 생겼습니다: 왼쪽에는 빈 작업 표, 오른쪽에는 빈 간트 차트, 그리고 기본 설정에서 이미 구성된 근무 달력이 있습니다.

## 2단계 — 작업 추가하기

리본 탭 **홈**에 있는지 확인하세요. 이 탭은 작업 표(왼쪽)와 간트 차트(오른쪽)를 나란히 보여줍니다 — 같은 일정의 두 가지 뷰이므로, 추가한 작업은 두 곳 모두에 동시에 나타납니다.

### 리본을 통해

1. **작업** 리본 그룹에서 **작업** 버튼을 클릭합니다. "새 작업"이라는 이름의 새 작업이 5 근무일의 기간으로 작업 표와 간트 차트 하단에 나타납니다.
2. 프로젝트의 주요 단계마다 작업이 생길 때까지 이를 몇 번 반복합니다. 예제 프로젝트를 따라간다면 동일한 주요 단계를 사용하세요: "1. Voorbereiding"(준비), "2. Fundering & ruwbouw"(기초 및 골조 공사), "3. Afbouw"(마감) 및 "4. Oplevering"(인도).
3. 작업을 더블클릭합니다 — 표나 간트 차트의 막대에서 — **작업 편집** 창을 엽니다. **이름**, **유형**, **기간 (근무일)**을 해당 단계에 맞게 조정합니다.

### 작업 표와 간트 차트를 통해

계속 리본으로 돌아갈 필요는 없습니다. 작업 표의 **빈 행**을 마우스 오른쪽 버튼으로 클릭하거나, 간트 차트의 빈 곳(아직 작업이 없는 곳)을 클릭하고, 컨텍스트 메뉴에서 **새 작업**을 선택하세요.

대신 **기존** 작업을 마우스 오른쪽 버튼으로 클릭하면, 다음을 포함한 다른 컨텍스트 메뉴가 나타납니다:

- **위에 삽입** / **아래에 삽입** — 마우스 오른쪽 버튼으로 클릭한 작업 앞이나 뒤에 작업을 추가합니다.
- **하위 작업 추가** — 한 단계로 해당 작업의 자식으로 새 작업을 만듭니다(이것이 무엇을 의미하는지는 3단계 참조).

무언가 잘못 입력했거나 작업을 잘못된 곳에 추가했나요? **Ctrl+Z**는 마지막 작업을 실행 취소하고, **Ctrl+Y**(또는 **Ctrl+Shift+Z**)는 다시 실행합니다 — 둘 다 텍스트 필드뿐만 아니라 전체 일정에서 작동합니다.

### 마일스톤 추가하기

모든 일정에는 최소 하나의 마일스톤이 필요합니다(예: 인도용). **작업** 리본 그룹에서 **마일스톤** 옆의 화살표를 클릭하고 **완료 마일스톤**, **시작 마일스톤** 또는 **검사 지점(필수)**를 선택하세요 — 또는 단축키 **Ctrl+M**을 사용하면 나중에 이름을 바꾸는 빠르고 일반적인 마일스톤("새 마일스톤")을 만들 수 있습니다.

## 3단계 — 작업 구조(WBS) 만들기

작업의 평면 목록은 금방 혼란스러워집니다. 작업을 들여쓰기하면 작업 구조(WBS)를 만들게 됩니다: 위의 작업이 자동으로 하위 작업의 전체 기간에 걸쳐 있는 **요약 작업**이 됩니다.

1. 다른 작업 아래에 놓아야 할 작업을 선택합니다 — 예를 들어 단계 작업 "2. Fundering & ruwbouw"(기초 및 골조 공사) 아래에 "Fundering aanbouw"(증축 기초).
2. **Alt+→**를 눌러 들여쓰기하거나, 마우스 오른쪽 버튼을 클릭하고 컨텍스트 메뉴에서 **들여쓰기**를 선택합니다. 위의 작업이 즉시 요약 작업으로 표시됩니다.
3. 너무 멀리 갔거나 작업을 최상위 레벨로 되돌리고 싶나요? **Alt+←**를 사용하거나, 마우스 오른쪽 버튼을 클릭하고 **내어쓰기**를 선택하세요.
4. 완전히 새로운 하위 작업에는 더 빠른 방법이 있습니다: 상위 작업을 마우스 오른쪽 버튼으로 클릭하고 **하위 작업 추가**를 선택하세요 — 그러면 별도의 추가 후 들여쓰기 단계를 건너뜁니다.

몇 레벨 깊이가 될 때까지 이를 반복하세요. 예제 프로젝트에서는 예를 들어 "2. Fundering & ruwbouw" 단계가 "Grondwerk aanbouw"(증축 토공사), "Fundering aanbouw"(증축 기초), "Begane grondvloer storten"(1층 바닥 타설), "Wanden opmetselen"(벽돌쌓기) 및 "Dakconstructie plaatsen"(지붕 구조 설치)라는 하위 작업으로 나뉩니다.

이 문서는 시작하기 위한 실무 수준의 WBS 구성만 다룹니다. 마일스톤 종류, 요약 작업, 작업 코드가 어떻게 함께 작동하는지 자세히 배우려면 가이드 [계획 및 WBS](docs://gids-plannen-wbs)를 읽어보세요.

## 4단계 — 관계 추가하기

관계가 없는 작업은 서로 독립적이며, 이전 작업을 변경해도 이동하지 않습니다. 관계(종속성)는 두 작업을 서로 연결합니다.

1. 연결하려는 두 작업의 막대가 간트 차트에 보이는지 확인합니다.
2. **Shift**를 누른 채 선행 작업의 막대에서 후속 작업의 막대로 드래그합니다. 놓는 즉시 지연 0 근무일의 **Finish-Start (FS)** 관계가 바로 생성됩니다 — 가장 흔한 관계입니다: 선행 작업이 끝나야만 후속 작업이 시작됩니다.
3. 놓은 직후 **관계 유형** 창이 나타납니다. 여기서 관계 유형(**FS**, **SS**, **FF** 또는 **SF**)을 변경하고 **지연**을 입력할 수 있습니다(예: 두 작업 사이 대기 시간 2 근무일의 경우 `2d`). 요약하면: **FS**(Finish-Start)는 선행 작업이 끝난 후 후속 작업이 시작하고, **SS**(Start-Start)는 두 작업이 (대략) 동시에 시작하며, **FF**(Finish-Finish)는 (대략) 동시에 끝나고, **SF**(Start-Finish)는 후속 작업이 끝나기 전에 선행 작업이 시작해야 합니다 — 마지막 유형은 건설 실무에서 가장 드뭅니다.
4. 드래그 없이 두 작업을 연결하고 싶으신가요? 리본 탭 **관계**로 이동하거나(또는 계획 탭의 **관계** 리본 그룹에서 **관리**를 클릭), 먼저 선행 작업을 선택한 다음 (Ctrl/Cmd를 누른 채) 후속 작업을 선택하고 **선택 항목으로 새 관계 만들기** 버튼을 사용하세요 — 이 버튼은 정확히 두 작업이 그 순서로 선택되었을 때만 작동합니다.

연습을 위해 최소 두 개의 관계를 추가하세요: 예를 들어 "1. Voorbereiding" → "2. Fundering & ruwbouw" 및 "2. Fundering & ruwbouw" → "3. Afbouw".

## 5단계 — 계산하기

이제 작업과 관계가 있으니, 일정을 계산할 수 있습니다(CPM — Critical Path Method).

1. **F5**를 누르거나, **일정** 리본 그룹에서 **계산** 버튼을 클릭합니다.
2. Open Planner Studio는 이제 모든 작업에 대해 가장 빠른/늦은 시작 및 종료 날짜, 여유, 그리고 어떤 작업이 크리티컬 패스에 있는지 계산합니다.
3. F5를 더 이상 신경 쓰고 싶지 않나요? **설정**에서 **자동 계산**을 켜세요. 그러면 F5를 수동으로 누르는 대신 일정이 오래되는 즉시 자동으로 다시 계산됩니다.

## 6단계 — 결과 읽기

- 화면 하단의 상태 표시줄에는 일정이 계산되면 예를 들어 "크리티컬 패스: 4개 작업, 62 근무일"이 표시됩니다. 마지막 계산 이후 무언가 변경했다면 대신 "오래됨 — 다시 계산하세요 (F5)"가 표시됩니다.
- 간트 차트에서 크리티컬 작업 — 여유가 없어 프로젝트 종료일을 직접 결정하는 작업 — 은 여유가 있는 작업과 다른 막대 색상을 갖습니다. 크리티컬 작업이 지연되면 전체 프로젝트 종료일이 함께 이동합니다; 여유가 있는 작업은 여유가 소진되지 않는 한 결과 없이 지연될 수 있습니다.
- 작업을 더블클릭하여 **작업 편집** 창을 다시 엽니다. **CPM 결과** 섹션 아래에서 작업별로 다음을 찾을 수 있습니다: **빠른 시작:**, **빠른 종료:**, **늦은 시작:**, **늦은 종료:**, **총 여유:**, **자유 여유:**, 그리고 작업이 **크리티컬 패스:**에 있는지 여부.
- 각 작업을 열지 않고도 이 데이터를 작업 표의 열로 보고 싶으신가요? 리본 탭 **보기**로 이동해서 **표시** 그룹에서 **열…**을 클릭하고 **크리티컬**과 **총 여유**를 체크하세요.

## 7단계 — 저장하기

1. **Ctrl+S**를 누르거나 **파일** 탭에서 **저장**을 클릭합니다. 처음에는 Open Planner Studio가 파일 이름과 위치를 묻습니다; 프로젝트는 네이티브 IFC 파일로 저장됩니다.
2. 다른 이름으로 사본을 보관하고 싶으신가요(예: 두 버전을 나란히 유지하려고)? **파일 → 다른 이름으로 저장**(단축키 **Ctrl+Shift+S**)을 사용하세요.

## 계속 연습하기

- 완전한 예제로 위 단계를 다시 재생해보세요: **파일 → 예제**를 통해 [Verbouwing & Aanbouw Eengezinswoning](examples://showcase-verbouwing-eengezinswoning.ifc)를 열고, 단계 사이의 FS 체인, 벽체와 지붕 작업 사이의 SS 겹침, 타일 및 도장 작업 사이의 FF 링크, 그리고 시작 전 허가 제약 조건(SNET)을 확인해보세요.
- 작업 구조, 요약 작업, 마일스톤 종류, 작업 코드에 대해 더 알고 싶으신가요? 가이드 [계획 및 WBS](docs://gids-plannen-wbs)를 읽어보세요.
- 화면 주요 영역을 시각적으로 둘러보고 싶으신가요? **보기** 탭 → **투어** 버튼을 통해, 또는 **파일** → **투어 시작**을 통해 투어를 다시 시작하세요.
