# معلومات المشروع

تحتوي نافذة **معلومات المشروع** على البيانات الوصفية للمشروع بالإضافة إلى قسم **الحساب** بخيارات الجدولة. يعمل النموذج نفسه أيضًا كمعالج المشروع لـ**جديد**.

## الفتح

- **الإعدادات** (علامة تبويب الشريط) ← مجموعة الشريط **المشروع** ← **معلومات المشروع**.
- نافذة الإعدادات (الترس ⚙) ← علامة التبويب **عام** ← **معلومات المشروع...**
- **ملف** ← **معلومات المشروع** — نسخة مبسّطة في Backstage، بحقول البيانات الوصفية فقط (بلا قسم الحساب).

**تطبيق** يثبّت كل التغييرات دفعة واحدة؛ **إلغاء** أو **Esc** أو نقرة خارج النافذة يُهملها. **Enter** يفعل ما يفعله تطبيق.

## البيانات الوصفية

- **اسم المشروع** — الاسم في شريط العنوان وعلامة تبويب المستند.
- **الوصف** — نص حر.
- **المهندس** و**الشركة** — نص حر؛ يُخزَّن في ملف IFC.
- **تاريخ البداية** — بداية المشروع التي يبدأ الحساب منها.
- **تاريخ النهاية** — نهاية المشروع الإخبارية.

## الحساب

خيارات الجدولة لهذا المشروع — تُخزَّن مع الملف، لا مع التطبيق، لذا تنتقل إلى أجهزة أخرى. إذا غيّرت أي شيء هنا، يُعاد حساب الجدول تلقائيًا بعد **تطبيق**.

- **تعريف الحرج** — **الفائض الكلي ≤ العتبة** (مع **العتبة (أيام عمل)**، الافتراضي 0) أو **المسار الأطول**.
- **حساب الفائض** — **الأصغر (البداية/النهاية)** (الافتراضي)، **فائض البداية** أو **فائض النهاية**.
- **المهام مفتوحة النهاية حرجة** — يعلّم المهام بلا لاحقة كحرجة.
- **وضع علامة شبه حرج** — تحديدها يكشف **العتبة** إضافية (الافتراضي يومَا عمل؛ الوحدة تتبع عرض المدة، فربما ساعات): المهام ذات الفائض القليل تحصل على علامة "شبه حرج".
- **مسارات فائض متعددة** — تحديدها يكشف **الطريقة** (**الفائض الحر (peeling)** أو **الفائض الكلي (ترتيب)**) و**الحد الأقصى للمسارات** (الافتراضي 10): يرقّم الحساب حينئذ أهم مسارات الفائض.
- **تقويم الإبطاء** — أي تقويم يحتسب إبطاء العلاقة: **السلف** (الافتراضي)، **الخلف**، **24 ساعة** أو **تقويم المشروع**.

كيفية قراءة هذه النتائج مشروحة في [المسار الحرج والتحليل المتقدم](docs://gids-kritiek-pad-analyse).

## معالج المشروع (جديد)

**جديد** يفتح النافذة نفسها كمعالج (العنوان **مشروع جديد**، الزر **إنشاء**). إلى جانب حقول البيانات الوصفية، يحتوي المعالج على:

- **قالب المراحل** — **فارغ**، **بناء سكني** أو **بناء تجاري / تجديد**: يملأ المشروع الجديد ببنية مراحل.
- **الوردية** — مرئية فقط مع تفعيل التخطيط بالساعات: **وردية نهارية** (الافتراضي)، **ورديتان**، **ثلاث ورديات** أو **24/7**.
- **مجموعة العطل** — تُنشئ تقويم المشروع: اختر دولة (مع منطقة وعطلة بناء حيث ينطبق)، **بدون عطل رسمية**، أو **مخصص…** — يفتح الأخير حوار التقويم مباشرة بعد الإنشاء لتكوين التقويم يدويًا. انظر [حوار التقويم](docs://ref-kalenderdialoog).

قسم الحساب غائب عن المعالج؛ اضبطه لاحقًا عبر أحد المداخل أعلاه.
