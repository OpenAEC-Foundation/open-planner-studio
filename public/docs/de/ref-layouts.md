# Layouts speichern und laden

Ein Layout ist eine gespeicherte Ansichtskonfiguration: die Spalten, Gruppierung, Sortierung, den Filter und die Zeitskala in einem Paket. Layouts sind App-global (auf diesem Gerät) — sie gehören nicht zu einer einzelnen Projektdatei, Sie können sie also in jedem Dokument verwenden.

## Öffnen

**Ansicht** → Menübandgruppe **Layout**. Sie hält eine Auswahl mit Ihren Layouts und drei Schaltflächen:

- **Speichern als…** und **Verwalten…** — beide öffnen das Fenster **Layouts verwalten** (unten).
- **Aktualisieren** — überschreibt das in der Auswahl gewählte Layout mit der aktuellen Ansicht; deaktiviert, solange **(keins)** gewählt ist.

Die Wahl eines Layouts in der Auswahl wendet es sofort an.

## Das Fenster Layouts verwalten

Ohne gespeicherte Layouts zeigt das Fenster „Noch keine gespeicherten Layouts." Sonst eine Zeile pro Layout mit:

- **Name** — direkt in der Zeile editierbar (umbenennen).
- **Übernehmen** (Häkchen) — fragt zuerst nach Bestätigung: „Layout „…" übernehmen? Dies ersetzt die aktuellen Spalten/Gruppierung/Sortierung/Filter/Skala."
- **Aktualisieren** — überschreibt das Layout mit der aktuellen Ansicht, ohne Bestätigung.
- **Löschen** (Papierkorb-Symbol) — fragt zuerst nach Bestätigung.

Die Bestätigungen erscheinen als kleiner In-App-Dialog; **Esc** oder **Abbrechen** bricht ab.

## Layout speichern als…

Unten im Fenster: Tippen Sie einen **Namen** und klicken Sie **Speichern** — die aktuelle Ansicht wird als neues Layout gespeichert und wird das aktive. Ohne Namen erhält das Layout den Standardnamen „Name".

## Was ein Layout erfasst

- Spalten (Sichtbarkeit, Reihenfolge, Breite) — siehe [Spalten wählen](docs://ref-kolommen).
- Gruppierung und Sortierung (**Ansicht** → **Gruppieren…** / **Sortieren…**).
- Den Filter — siehe [Filter](docs://ref-filters).
- Die Gantt-Zeitskala.

Nicht enthalten: Zoomstufen-Details, Bereichsbreiten und Auswahlen.
